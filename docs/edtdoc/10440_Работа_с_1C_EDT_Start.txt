## Запустить 1C:EDT Start

Запустить 1C:EDT Start можно разными способами:
- С помощью ярлыка 1CEDT Start на рабочем столе (Windows и Linux);Примечание: Для Linux создано большое количество оболочек (desktop environment), несовместимых между собой. Это сказывается на механизме создания ярлыков для установленного программного обеспечения. Программа установки ориентируется на спецификацию Desktop Menu Specification от FreeDesktop — подробнее — и ее работа проверена на Gnome 3 и KDE 5.

- С помощью команды Пуск > 1C Enterprise Development Tools > 1CEDT Start (Windows) или с помощью команды 1CEDT Start в Launchpad (macOS);

- Можно запустить исполняемый файл 1cedtstart из домашней директории установки.При стандартной установке этот файл находится в следующих каталогах (версия вашей 1C:EDT Start может отличаться от указанной здесь версии):

Windows%ProgramFiles%\1C\1CE\components\1c-edt-start-0.6.0+244-x86_64\1cedtstart.exe

Linux/opt/1C/1CE/components/1c-edt-start-0.6.0+244-x86_64/1cedtstart

macOS/Applications/1C/1CE/components/1c-edt-start-0.6.0+244-x86_64/1cedtstart (0.6.0+244).app

## Скрыть и открыть 1C:EDT Start

Скрыть 1C:EDT Start

Кнопка закрытия окна сворачивает 1C:EDT Start в панели задач. Это ее нормальное состояние.

Открыть 1C:EDT Start

Чтобы открыть свернутую 1C:EDT Start, нажмите Открыть в ее контекстном меню.

## Завершить работу 1C:EDT Start

В панели задач нажмите Выйти в контекстном меню 1C:EDT Start.

## Работа через корпоративный прокси-сервер

1C:EDT Start может работать через корпоративный прокси-сервер на ОС Windows и Linux. Для этого достаточно, чтобы параметры прокси-сервера были правильно указаны в настройках операционной системы.

Windows

Чтобы настроить параметры прокси-сервера на Windows 10 нажмите Пуск > Параметры > Сеть и Интернет > Прокси-сервер.

После изменения этих параметров перезапустите 1C:EDT Start.

Linux

На Ubuntu 18.04 параметры прокси-сервера находятся в следующих переменных окружения:

- HTTP_PROXY

- http_proxy

- NO_PROXY

- no_proxy

- HTTPS_PROXY

- https_proxy

Чтобы проверить их значения выполните:

env | grep -i proxy
Чтобы настроить параметры прокси-сервера нажмите Параметры > Сеть > Сетевой прокси.

После изменения этих параметров перезапустите 1C:EDT Start.

Индивидуальные настройки прокси-сервера

Если требуются настройки прокси, отличные от системных, или если нужно использовать корпоративный прокси с аутентификацией, эти настройки можно задать следующим образом:

- Откройте настройки 1C:EDT Start;

- Прокрутите список вниз до раздела Настройки прокси;

- Установите переключатель Указать HTTP прокси вручную;

- Задайте значения Хост, Порт, Логин и Пароль;

- Закройте настройки 1C:EDT Start;

## Авторизация на портале 1С:ИТС

Авторизация на портале ИТС необходима для того, чтобы вы могли автоматически устанавливать среды разработки и получать обновления для них.

О том, что вы авторизованы на портале 1С:ИТС, говорит значок в заголовке 1C:EDT Start:

- Есть авторизация ;

- Нет авторизации .

В любой момент можно нажать на этот значок, чтобы авторизоваться или, наоборот, выйти из учетной записи на портале 1С:ИТС.

1C:EDT Start запрашивает авторизацию автоматически, при запуске. Если у вас отсутствует подключение к интернету, или если вы не можете авторизоваться на портале 1С:ИТС, можно отказаться от авторизации.

В этом случае вы продолжите работу, но без авторизации.

Работа без авторизации на портале 1С:ИТС вполне приемлема в тех случаях, например, когда вы не имеете доступа в интернет. Однако этом случае вы не сможете:

- Автоматически устанавливать среды разработки (подробнее. Чтобы установить новую среду разработки вам нужно будет получить дистрибутив для установки без интернета на каком-то носителе и установить его;

- Автоматически обновлять среды разработки (подробнее). Обновить установленную среду разработки можно будет только установив новую версию среды отдельно, из дистрибутива для установки без интернета. При этом вам нужно будет перенести в новую версию те настройки, плагины и пр., которые существовали в старой версии.

## Размер и положение 1C:EDT Start

Можно менять размер и положение окна 1C:EDT Start:
- Можно растягивать окно по вертикали;

- Двойным кликом на заголовке окна можно раскрывать его на всю высоту экрана и уменьшать до минимальной высоты при повторном двойном клике;

- Расположение окна на экране и экран, если их несколько, запоминаются при последующих запусках 1C:EDT Start;

- Высота окна тоже запоминается при последующих запусках 1C:EDT Start.

## Уведомления

1C:EDT Start оповещает о выходе новых версий среды разработки:

- Иконка приложения в панели задач начинает отображаться с красной точкой ;

- В заголовке 1C:EDT Start отображается колокольчик , на котором показано количество непрочитанных уведомлений.

Открыть все уведомления

Нажмите колокольчик в заголовке 1C:EDT Start.

Обновить релиз из уведомления

Если это уведомление о новой версии установленного релиза, то в нем будет кнопка Обновить. Нажмите ее, чтобы обновить релиз.

Удалить одно уведомление

Нажмите крестик в уведомлении.

Удалить все уведомления

Нажмите Очистить все.

## Настройки 1C:EDT Start

Открыть настройки

Нажмите шестеренку в заголовке.

Закрыть настройки

Нажмите крестик в заголовке. Измененные значения будут сохранены автоматически.

## Темная тема

1C:EDT Start может использовать светлую или темную тему интерфейса. Стандартно, после
установки, включена светлая тема.

Чтобы включить темную тему, выполните следующие действия:

- Откройте настройки 1C:EDT
Start;

- В поле Основные > Тема приложения включите выключатель;

- Закройте настройки 1C:EDT
Start;

Чтобы вернуться к светлой теме, выключите этот же выключатель.

Как включить темную тему в 1C:EDT, читайте
здесь.

## Изменить путь к проектам

- Откройте настройки 1C:EDT Start;

- В поле Основные > Путь к проектам введите новый путь;

- Закройте настройки 1C:EDT Start;

## Изменить путь к средам разработки

Чтобы изменить каталог, в котором стандартно хранятся среды разработки, выполните следующие действия:

- Откройте настройки 1C:EDT Start;

- В поле Основные > Путь к средам разработки введите новый путь;

- Закройте настройки 1C:EDT Start;

## Версия программы 1C:EDT Start

Чтобы узнать версию программы 1C:EDT Start, выполните следующие действия:

- Откройте настройки программы;

- Нажмите О программе;

## Журнал приложения

Для анализа неправильной работы 1C:EDT Start технической поддержке могут понадобиться ..log-файлы. Чтобы открыть каталог с этими файлами, выполните следующие действия:

- Откройте настройки 1C:EDT Start;

- В строке Основные > Журнал приложения нажмите значок .

## Сбор и передача статистики

В 1C:EDT автоматически собирает статистику возникающих ошибок и статистику использования той или иной функциональности. Эта информация объединяется и отправляется на сервис мониторинга для анализа. Сервис мониторинга расположен на инфраструктуре фирмы «1С» (подробнее).

Стандартно этот механизм включен, но его можно выключить. Для этого:

- Откройте настройки 1C:EDT Start;

- Пролистайте настройки вниз и выключите переключатель Основные > Включить сбор и передачу статистики.

- Закройте настройки 1C:EDT Start.

## Общее хранилище плагинов

Стандартно 1C:EDT Start, при установке сред разработки, сохраняет плагины в общем для
всех сред разработки хранилище. Таким образом, если у двух сред разработки одна и та
же версия одного плагина, то они будут использовать общий плагин из этого
хранилища.

Вы можете выключить общее хранилище плагинов и тогда все плагины для вновь
установленной среды разработки будут сохраняться в каталог этой среды разработки.
При удалении среды разработки все ее плагины так же удалятся и освободят место на
диске.

Чтобы отключить использование общего хранилища плагинов, выполните следующие
действия:

- Откройте настройки 1C:EDT
Start;

- В поле Дополнительно > Использовать общее хранилище плагинов выключите выключатель;

- Закройте настройки 1C:EDT
Start;

## Установка для всех пользователей

Стандартно 1C:EDT Start устанавливает среды разработки для текущего пользователя, от имени которого он запущен.

Если на компьютере работают разные пользователи, которым нужны одни и те же среды разработки, можно устанавливать их для всех пользователей, один раз. Для этого:

- Откройте настройки 1C:EDT Start;

- В поле Основные > Путь к средам разработки укажите папку, которая доступна всем пользователям операционной системы.В результате другие пользователи будут иметь доступ к установленным средам разработки;

- Пролистайте настройки до раздела Дополнительно;

- Включите переключатель Установка для всех пользователей;В результате информация об устанавливаемых средах разработки будет сохраняться в общий реестр сред разработки и 1C:EDT Start, запущенные от имени других пользователей, будут ее видеть.

- Закройте настройки 1C:EDT Start.

Среды разработки, установленные после этого, будут доступны всем пользователям операционной системы.
