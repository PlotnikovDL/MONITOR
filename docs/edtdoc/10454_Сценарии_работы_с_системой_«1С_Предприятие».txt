## Запустить информационную базу в режиме «Конфигуратор»

В режиме Конфигуратор выполняется разработка приложений и администрирование информационных баз.

Чтобы запустить информационную базу в режиме Конфигуратор, выполните следующие действия:

- Запустите платформу «1С:Предприятие»;

- Выберите информационную базу, с которой вы будете работать;

- Нажмите Конфигуратор...;

Возможная проблема: Если запуск информационной базы в этом режиме выполняется в первый раз, структура конфигурации в конфигураторе будет скрыта. Чтобы открыть ее нажмите Конфигурация > Открыть конфигурацию.

## Запустить информационную базу в режиме «1С:Предприятие»

Режим 1С:Предприятие — это основной режим системы «1С:Предприятие». В нем работают пользователи (в отличие от администраторов и разработчиков). В этом режиме пользователи добавляют, изменяют, удаляют данные, формируют отчеты и выполняют другие прикладные задачи.

Чтобы запустить информационную базу в режиме 1С:Предприятие, выполните следующие действия:
- Запустите платформу «1С:Предприятие»;

- Выберите информационную базу, с которой вы будете работать;

- Нажмите 1С:Предприятие.

## Создать файловую информационную базу без конфигурации

Чтобы создать файловую информационную базу без конфигурации, выполните следующие действия:
- Запустите платформу «1С:Предприятие»;

- Нажмите Добавить...;

- Выберите пункт Создание новой информационной базы, нажмите Далее;

- Выберите пункт Создание информационной базы без конфигурации ..., нажмите Далее;

- Задайте имя информационной базы, по которому вы будете отличать ее от других информационных баз на вашем компьютере;

- Выберите пункт На данном компьютере или на компьютере в локальной сети, нажмите Далее;

- Нажмите Далее, затем Готово.

## Создать файловую информационную базу из шаблона

Чтобы создать файловую информационную базу из шаблона, выполните следующие действия:
- Запустите платформу «1С:Предприятие»;

- Нажмите Добавить...;

- Выберите пункт Создание новой информационной базы, нажмите Далее;

- Выберите пункт Создание информационной базы из шаблона, выберите шаблон, нажмите Далее;

- Задайте имя информационной базы, по которому вы будете отличать ее от других информационных баз на вашем компьютере;

- Выберите пункт На данном компьютере или на компьютере в локальной сети, нажмите Далее;

- Нажмите Далее, затем Готово.

## Загрузить конфигурацию из .cf-файла

В этом сценарии предполагается, что информационная база, в которую будет загружен .cf-файл, либо не содержит конфигурацию, либо содержит конфигурацию и данные, которые больше не нужны и вы готовы их удалить.

Важно: Для выполнения этого сценария информационная база должна быть запущена в режиме Конфигуратор.

Чтобы загрузить конфигурацию из .cf-файла, выполните следующие действия:
- Нажмите Конфигурация > Загрузить конфигурацию из файла...;

- Выберите .cf-файл, нажмите Открыть.Если информационная база уже содержит конфигурацию, конфигуратор предупредит об этом;

- Нажмите Да. Конфигурация из .cf-файла будет загружена в информационную базу как основная конфигурация. Конфигуратор задаст вопрос об обновлении конфигурации базы данных;

- Нажмите Да; Конфигуратор сообщит о том, что будет выполнена реструктуризация базы данных;

- Нажмите Принять.Конфигурация базы данных будет обновлена.

## Загрузить информационную базу из .dt-файла

В этом сценарии предполагается, что информационная база, в которую будет загружен .dt-файл, либо не содержит конфигурацию, либо содержит конфигурацию и данные, которые больше не нужны и вы готовы их удалить.

Важно: Для выполнения этого сценария информационная база должна быть запущена в режиме Конфигуратор.

- Нажмите Администрирование > Загрузить информационную базу...:

- Выберите .dt-файл, нажмите Открыть.Будет открыто предупреждение о том, что работа конфигуратора будет завершена;

- Нажмите Да. Информационная база будет загружена. Будет показан вопрос о перезапуске конфигуратора;

- Нажмите Да. Конфигуратор будет перезапущен с новой информационной базой.

## Создать .cf-файл поставки конфигурации

Важно: Для выполнения этого сценария информационная база должна быть запущена в режиме Конфигуратор.

Чтобы создать .cf-файл поставки конфигурации выполните следующие действия:
- Нажмите Конфигурация > Поставка конфигурации > Создать файлы поставки и обновления конфигурации...;

- Установите флажок Создать файл поставки;

- Укажите расположение .cf-файла поставки, который будет создан, и нажмите Выполнить;

- Нажмите ОК.

## Создать .cfu-файла обновления конфигурации

Важно: Для выполнения этого сценария информационная база должна быть запущена в режиме Конфигуратор.

Чтобы создать .cfu-файл обновления конфигурации выполните следующие действия:
- Нажмите Конфигурация > Поставка конфигурации > Создать файлы поставки и обновления конфигурации...:

- Установите флажок Создать файл обновления конфигурации;

- Укажите расположение .cfu-файла обновления, который будет создан, и нажмите Выполнить;

- В список Конфигурации для обновления добавьте .cf-файл поставки предыдущей версии вашего прикладного решения;

- Нажмите Выполнить:

- Нажмите ОК.

## Обновить конфигурацию с помощью .cfu-файла

Важно: Для выполнения этого сценария информационная база должна быть запущена в режиме Конфигуратор.

Чтобы обновить конфигурацию с помощью .cfu-файла, выполните следующие действия:
- Нажмите Конфигурация > Поддержка > Обновить конфигурацию...;

- Выберите пункт Выбор файла обновления, нажмите Далее;

- Выберите .cfu-файл, нажмите Готово;

- Нажмите ОК. Будет выполнено обновление основной конфигурации. Будет показан вопрос об обновлении конфигурации базы данных;

- Нажмите Да. Будет показана информация о реструктуризации базы данных;

- Нажмите Принять.

## Создать комплект поставки из текущей информационной базы

Важно: Для выполнения этого сценария информационная база должна быть запущена в режиме Конфигуратор.

Чтобы создать комплект поставки из текущей информационной базы, выполните следующие действия:

- Нажмите Конфигурация > Поставка конфигурации > Комплект поставки...;

- Выберите пункт Создать новое описание комплекта поставки, нажмите ОК;

- Дважды нажмите Далее, а затем Готово;

- Нажмите Создать архив комплекта;

- Нажмите Да;

- Выберите каталог, в который будет сохранено описание комплекта поставки, нажмите Сохранить;

- Выберите каталог, в который будет сохранен архив комплекта поставки, нажмите Сохранить.

## Выгрузить конфигурацию в файлы

Важно: Для выполнения этого сценария
информационная база должна быть запущена в
режиме Конфигуратор.

- Нажмите Конфигурация > Выгрузить конфигурацию в файлы...;
- Выгрузить в — выберите один из вариантов выгрузки. Для последующего
импорта в 1C:EDT подойдут оба варианта:
- XML-файлы — выберите этот вариант, если
нужно выгрузить конфигурацию в иерархическую структуру
.xml-файлов, и укажите путь к каталогу с будущими файлами;

-
Архив — выберите этот вариант, если нужно
дополнительно поместить все выгруженные файлы в архив, и укажите
путь к будущему файлу архива;

- В поле ввода выберите каталог, в который будут помещены файлы выгрузки
или архив;

- Выполнять на сервере — если конфигуратор запущен
для клиент-серверной информационной базы, то этот флажок доступен и с
его помощью можно указать, что выгрузка будет выполняться на сервере. В
этом случае будут использоваться несколько системных фоновых
заданий;

- Количество заданий выгрузки — если 0, то
платформа «1С:Предприятие» автоматически определит количество системных
фоновых заданий для выгрузки клиент-серверной базы. Иначе вы можете
указать количество этих заданий;

- Не выгружать ссылки на несуществующие объекты — в
структуре конфигурации могут существовать ссылки на такие объекты
конфигурации, которые были когда-то из нее удалены. Эти ссылки ведут в
никуда и в идеальной конфигурации таких ссылок быть не должно. Установка
этого флажка позволяет не выгружать такие ссылки.

- Нажмите Выгрузить.

За ходом выгрузки можно наблюдать в панели состояния, расположенной в нижней части
конфигуратора. По окончании выгрузки никаких информационных сообщений не
выдается.

## Загрузить конфигурацию из файлов

Важно: Для выполнения этого сценария
информационная база должна быть запущена в
режиме Конфигуратор.

- Нажмите Конфигурация > Загрузить конфигурацию из файлов...;
- Загрузить из — выберите один из вариантов выгрузки. Для последующего
импорта в 1C:EDT подойдут оба варианта:
- XML-файлы — выберите этот вариант, если
вы выгружали проект в .xml-файлы;

-
Архив — выберите этот вариант, если вы
выгружали проект в архив;

- В поле ввода выберите каталог, в котором находятся файлы выгрузки или
архив;

- Нажмите Загрузить.

За ходом загрузки можно наблюдать в панели состояния, расположенной в нижней части
конфигуратора. По окончании загрузки никаких информационных сообщений не
выдается.
