## Все среды разработки

Открыть среды разработки

- Нажмите шестеренку в заголовке 1C:EDT Start;

- Нажмите строку в разделе Среды разработки;

В разделе Установлено перечислены установленные релизы с точностью до третьего разряда (патча).

Версии, установленные без интернета, также перечислены здесь. Они помечены треугольником , который означает, что обновление этих релизов средствами 1C:EDT Start невозможно.

В разделе Доступно для установки для каждой из возможных сред разработки перечислены релизы, которые вы можете установить. Такие строки содержат стрелку вправо . Если вы нажмете на такую строку, то перейдете к списку релизов, в котором можно выбрать один из них.

Если для установки доступен единственный релиз, то такая строка содержит стрелку вниз . Если вы нажмете на такую строку, то среда разработки начнет скачиваться и устанавливаться.

## Установить новый релиз среды разработки

- Откройте среды разработки;

- В разделе Доступно для установки нажмите стрелку вправо , чтобы открыть список доступных релизов;

- Выберите нужный релиз;

- 1C:EDT Start переключится на список сред разработки и начнет скачивание и установку нового релиза. О ходе процесса вас информирует индикатор.

Не забудьте: Настройте параметры запуска нового релиза.

После того, как среда разработки будет установлена, вы можете назначить ее существующему проекту.

Ловкий прием: Если вы сразу же хотите использовать для проекта новый, еще не установленный релиз среды разработки, выберите его в настройках проекта. Он будет автоматически скачан и установлен при запуске проекта.

## Обновить среду разработки

1C:EDT Start оповещает о выходе новых версий. Если вышла новая версия релиза, который установлен у вас, можно автоматически обновить его несколькими способами.

Обновить из уведомления

- Откройте уведомления;

- В уведомлении о выходе нужной версии нажмите Установить.

Обновить из среды разработки

- Откройте среды разработки;

- Наведите курсор на версию, у которой есть надпись Что нового ... и нажмите закрученную стрелку .

Установленный релиз будет обновлен, все проекты, которые используют его, будут запускаться на новой версии этого релиза.

## Настройки среды разработки

Открыть настройки среды разработки

- Откройте среды разработки;

- Наведите курсор на нужную установленную среду разработки и нажмите шестеренку .

Закрыть настройки среды разработки

Нажмите стрелку или крестик в заголовке. Измененные значения будут сохранены автоматически.

## Изменить параметры запуска среды разработки

Откройте настройки среды разработки.

Путь к виртуальной машине Java

В поле Java VM укажите путь к виртуальной машине Java. Если вы
устанавливали дистрибутив Axiom JDK стандартным образом, то
нужно указать такие пути:

WindowsC:\Program Files\Axiom\AxiomJDK-Pro-17-Full\bin

Linux/usr/lib/jvm/axiomjdk-java17-pro-full-amd64/bin

macOS/Library/Java/JavaVirtualMachines/axiomjdk-jdk-pro-17-full.jdk/Contents/Home/bin

Максимальная выделенная память

Импорт больших типовых конфигураций вроде 1C:ERP может занять длительное время и потребовать значительного количества оперативной памяти. Поэтому, если вы планируете выполнять такой импорт, увеличьте количество памяти, доступной виртуальной машине Java.

В поле Максимальная выделенная память выберите значение 8 Gb.

Расположение временного каталога

Большие типовые конфигурации могут содержать объекты с длинными именами.

Для загрузки в 1C:EDT конфигурация сначала выгружается из информационной базы во временный каталог в иерархию папок. Эта иерархия повторяет структуру конфигурации. В результате полный путь к какому-нибудь .xml-файлу конфигурации может превысить ограничения, накладываемые операционной системой, и импорт конфигурации закончится ошибкой.

Чтобы избежать этого, поместите временный каталог, который 1C:EDT использует для импорта и экспорта файлов конфигурации, как можно ближе к корню диска.
- Нажмите Отредактировать как параметры Java VM...;

- В поле Аргументы Java VM добавьте строку:-Djava.io.tmpdir=C:\tmpгде вместо C:\tmp укажите каталог временных файлов, который вы хотите использовать.

Совет: Подробнее об аргументах JVM, которые могут вам
понадобиться, читайте
здесь.

Язык интерфейса

1C:EDT использует для своего интерфейса языковые настройки операционной системы.

Если вы используете англоязычную операционную систему и хотите работать с 1C:EDT используя русский язык, в поле Язык интерфейса выберите значение Русский.

## Удалить среду разработки

- Откройте среды разработки;

- Наведите курсор на установленную среду разработки и нажмите корзину ;

- Нажмите Подтвердить.

На уровень выше: Работа с программой запуска и обновления 1C:EDT Start
