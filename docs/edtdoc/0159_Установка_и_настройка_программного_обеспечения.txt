# Установка и настройка программного обеспечения

Вам понадобится установить и настроить программное обеспечение на четырех компьютерах (группах):

- Сервер хранилища
конфигурации;

- Сервер «1С:ГитКонвертера»;

- Git-сервер;

- Компьютеры
разработчиков.

## Настройка сервера хранилища конфигурации

Для целей конвертации хранилища и синхронизации его с репозиторием мы рекомендуем
использовать сервер хранилища конфигурации. По нашему опыту в данной ситуации он
работает намного быстрее простого файлового хранилища.

Убедитесь, что версия 1С:Предприятия, используемая для работы
с хранилищем, поддерживается той версией
1C:EDT (1.8.1 и выше), которую вы будете устанавливать на сервер
1С:ГитКонвертера. Если это не так, конвертируйте
хранилище конфигурации на нужную версию платформы.

Если вы решили использовать копии хранилища, настройте размер глобального кэша так,
как это описано в разделе Копии
хранилища.

## Настройка сервера «1С:ГитКонвертера»

Настройка сервера «1С:ГитКонвертера» состоит из трех частей:

- Установка программного
обеспечения

- Настройка конфигурации 1С:ГитКонвертер

- Создание локального репозитория

## Настройка Git-сервера

В качестве сервера Git вы можете использовать одну из публичных систем управления
репозиториями, которые есть в интернете. Например, GitLab, GitHub, BitBucket и др.

Их достоинство заключается в том, что от вас не требуется предварительная установка и
настройка программного обеспечения.

Недостатком является то, что для размещения удаленного репозитория вам, скорее всего,
придется использовать платный аккаунт.

В то же время, например, GitLab имеет бесплатную версию, которую вы можете скачать и
установить на сервере своей организации.

## Настройка Git на компьютерах разработчиков

Описанные ниже настройки необходимо выполнить на каждом компьютере тех разработчиков,
которые будут вести групповую разработку технического проекта в 1C:EDT.
- Установите 1C:EDT той же версии, которая установлена на сервере
1С:ГитКонвертера. Совпадение версий важно, так
как номер версии 1C:EDT, с помощью которой редактируется конфигурация,
сохраняется в проекте 1C:EDT;

- Если при конвертации хранилища в репозиторий вы решили использовать Git LFS,
установите и настройте Git LFS на компьютере разработчика так, как это
описано в разделе Git LFS;

- Настройте Git для совместной работы так, как это описано в документации
1C:EDT в разделе Настройки
групповой разработки.

- Настройка сервера «1С:ГитКонвертера»

На уровень выше: Методика постепенного перехода на разработку в 1C:EDT
