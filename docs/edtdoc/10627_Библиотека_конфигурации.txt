## Настроить правила поставки

- В панели Навигатор в контекстном меню конфигурации нажмите
Настройка поставки....

- Раскройте дерево объектов, выделите правила, которые вы хотите изменить.

- В контекстном меню нажмите Установить правило
поставки.

- Выберите новое правило и нажмите ОК.

- Сохраните изменения, выполненные в редакторе.

## Экспортировать проект в файл библиотеки

- В панели Навигатор в контекстном меню корня проекта
нажмите Экспорт... > 1C:Enterprise > Конфигурация в файл библиотеки конфигурации.

- Задайте свойства библиотеки:
- Версия библиотеки, например,
1.0.38.2.

- Поставщик библиотеки, например, ООО
"Ромашка".

- Файл библиотеки, например,
C:\edt\Lib\BaseApp-1.0.38.2.

- Нажмите Готово. По указанному пути будет создан файл
библиотеки BaseApp-1.0.38.2.clib.

## Экспортировать новую версию библиотеки

- В панели Навигатор в контекстном меню корня проекта
нажмите Экспорт... > 1C:Enterprise > Конфигурация в файл библиотеки конфигурации.

- Задайте свойства библиотеки:
- Версия библиотеки — увеличьте номер версии,
например, 1.0.38.3.

- Файл библиотеки, например,
C:\edt\Lib\BaseApp-1.0.38.3.

- Нажмите Готово. По указанному пути будет создан файл
библиотеки BaseApp-1.0.38.3.clib.

## Создать проект на основе библиотеки

- В панели Навигатор нажмите Создать новую
конфигурацию.

- Задайте свойства конфигурации:
- Тип конфигурации выберите Конфигурация
на основе библиотеки.

- Файл библиотеки выберите .clib-файл. В поле
Библиотека появится представление выбранной
библиотеки.

- Имя проекта задайте имя создаваемого проекта.

- Нажмите Готово. Будет создан библиотечный проект, изменение которого
запрещено. О том, что изменение запрещено, говорит значок замка на конфигурации, а также недоступные
команды создания объектов.

## Включить возможность изменения проекта

- В панели Навигатор в контекстном меню конфигурации нажмите
Настройка поддержки....

- В редакторе правил поддержки нажмите на синий замок Включить
возможность изменения.

- Согласитесь с предупреждением и нажмите ОК.

- Измените, или нет, правила поддержки библиотечных объектов:
- Если вы хотите только добавлять новые объекты в проект или изменять
его командный интерфейс, нажмите ОК.

- Если вы хотите изменять все библиотечные объекты, для которых
разработчик допускает изменение, выберите подходящие правила
поддержки и нажмите ОК. Эти правила поддержки
будут установлены для всех объектов с указанными правилами
поставки.

- Если вы хотите изменять только некоторые библиотечные объекты,
нажмите ОК. После этого измените правила
поддержки только для нужных вам объектов.

## Изменить правила поддержки

Чтобы изменять правила поддержки, должна быть включена возможность
изменения проекта.

- В панели Навигатор в контекстном меню конфигурации нажмите
Настройка поддержки....

- Найдите нужный вам объект и двойным кликом на правиле поддержки измените
его.

- Сохраните изменения, выполненные в редакторе.

## Обновить библиотеку в проекте

- В панели Навигатор в контекстном меню корня проекта
нажмите Обновить библиотеку конфигурации....

- В поле Из файла библиотеки выберите .clib-файл. В полях
диалога появится информация о выбранной библиотеке.

- Нажмите Готово. 1C:EDT откроет редактор сравнения и
объединения.

- Разрешите конфликты, если они есть, и нажмите
Объединить.

## Подключить библиотеку к существующему проекту

- В панели Навигатор в контекстном меню корня проекта
нажмите Подключить библиотеку конфигурации....

- В поле Из файла библиотеки выберите .clib-файл. В полях
диалога появится информация о выбранной библиотеке.

- Нажмите Готово. 1C:EDT откроет редактор сравнения и
объединения.

- Разрешите конфликты, если они есть, и нажмите
Объединить.

## Подключить проект к нескольким библиотекам

- Создайте проект
на основе библиотеки.

- Включите возможность изменения библиотечного проекта.

- Подключите вторую библиотеку к проекту.
