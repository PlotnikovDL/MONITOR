# Обновление с помощью 1C:EDT Start

- Обновление релиза

- Перевод проекта на новый релиз

1C:EDT Start различает версии 1C:EDT с точностью до релиза, то есть до второго разряда версии.

Например, версии 2022.1.3 и 2022.1.5 это один и тот же релиз 2022.1. Поэтому с помощью 1C:EDT Start на компьютере не могут быть одновременно установлены обе эти версии. Если при установленной 2022.1.3 вы захотите установить 2022.1.5, то прежняя версия будет обновлена на новую. Соответственно все проекты, использовавшие версию 2022.1.3, станут работать на версии 2022.1.5. Эта операция называется обновлением релиза.

В то же время с помощью 1C:EDT Start два разных релиза можно установить на один компьютер, например 2022.1 и 2022.2. Если при установленной версии 2022.1.3 вы захотите установить версию 2022.2.5, то прежняя версия затронута не будет. Таким образом одни проекты могут использовать версию 2022.1.3, а другие проекты — версию 2022.2.5. Эта операция называется установкой нового релиза.

Примечание: При использовании дистрибутива для установки без интернета каждая версия устанавливается отдельно, таким образом вы можете одновременно использовать, например, версии 2022.1.3 и 2022.1.5.

## Обновление релиза

1C:EDT Start оповещает о выходе новых версий. Если вышла новая версия релиза, который установлен у вас, можно автоматически обновить его несколькими способами.

Обновить из уведомления

- Откройте уведомления;

- В уведомлении о выходе нужной версии нажмите Установить.

Обновить из среды разработки

- Откройте среды разработки;

- Наведите курсор на версию, у которой есть надпись Что нового ... и нажмите закрученную стрелку .

Установленный релиз будет обновлен, все проекты, которые используют его, будут запускаться на новой версии этого релиза.

## Перевод проекта на новый релиз

- Откройте настройки проекта;

- Прокрутите список вниз до раздела Среда разработки;

- Нажмите карандаш у используемой среды разработки;

- Выберите установленный или доступный для установки релиз среды разработки;

- Закройте настройки проекта.

Если вы выбрали неустановленную среду разработки, она будет скачана и установлена при запуске проекта.

На уровень выше: Установка, обновление
