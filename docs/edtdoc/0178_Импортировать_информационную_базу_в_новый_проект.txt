# Импортировать информационную базу в новый проект

Если у вас есть приложение, которые вы создавали или модифицировали с помощью конфигуратора «1С:Предприятия», то продолжить его разработку можно в 1C:EDT. Этот пример показывает, как импортировать конфигурацию из базы в новый проект.

Важно: Этот пример предполагает, что вы уже выполнили первый запуск 1C:EDT и настроили среду разработки.

Возможная проблема: Перед выполнением примера убедитесь, что на компьютере установлена та версия платформы «1С:Предприятие», которая необходима для работы с импортируемой базой.

- Запустите проект 1C:EDT Start.

- Нажмите (Информационные базы) в свернутой панели справа.

1C:EDT раскроет панель Информационные базы;

- Далее нужно будет указать пользователя импортируемой базы, от имени которого 1C:EDT будет взаимодействовать с этой базой. Если вы не помните, какие пользователи есть в базе, запустите конфигуратор «1С:Предприятия». Для этого нажмите Запустить Конфигуратор в контекстном меню базы.

- Перетащите выбранную базу в панель Навигатор.

1C:EDT запустит мастер Импорт конфигурации, а затем откроет окно Сконфигурируйте доступ к информационной базе;

- В окне Сконфигурируйте доступ к информационной базе укажите настройки доступа, которыми будет пользоваться 1C:EDT при взаимодействии с этой базой:

- Тип доступа: Infobase;

- Имя пользователя — имя пользователя, обладающего административными правами;

- Пароль - пароль пользователя (если есть).

Нажмите ОК.

- 1C:EDT проанализирует базу и покажет расширения конфигурации, которые можно загрузить.Стандартно 1C:EDT предложит использовать для нового проекта самую старшую версию платформы «1С:Предприятие» из тех, что установлены на компьютере. Если для вашей базы нужна другая версия — выберите ее.

- Нажмите Импорт.1C:EDT начнет импорт конфигурации и расширений.

В результате в панели Навигатор будет открыт новый проект, содержащий импортированную конфигурацию. Рядом с ним будут открыты проекты расширений (на каждое расширение свой проект). Их названия в качестве префикса будут содержать имя родительского проекта.
