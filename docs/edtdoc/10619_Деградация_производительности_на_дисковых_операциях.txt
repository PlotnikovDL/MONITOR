# Деградация производительности на дисковых операциях

При работе с 1C:EDT на терминальных серверах (с использованием служб Windows RDS) или при
непосредственной работе на Windows-компьютере в некоторых ситуациях может наблюдаться
большая деградация производительности дисковых операций. Например, во время клонирования
репозитория Git, во время полного построения проекта или сравнения проекта с веткой Git.

Причиной этого может быть функция Disk Fair Share, включенная в службе RDS. Ее назначение
заключается в том, чтобы предотвращать чрезмерное использование дисков одним
пользователем, равномерно распределяя дисковые операции I/O между сессиями.

В некоторых операционных системах, например, Windows Server 2019, эта функция включена
по-умолчанию, в то время как на других ОС (Windows 10 Корпоративная, Windows 11
Корпоративная) она по-умолчанию выключена.

При наличии проблем с производительностью дисковых операций мы рекомендуем отключить Disk
Fair Share. Для этого выполните следующие команды:

$temp = (gwmi win32_terminalservicesetting -N "root\cimv2\terminalservices")
$temp.enableDiskFSS = 0
$temp.put()
Подробнее:

- Fair Share technologies are enabled by default in
Remote Desktop Services

- WMI properties: Win32_TerminalServiceSetting
class: Properties
