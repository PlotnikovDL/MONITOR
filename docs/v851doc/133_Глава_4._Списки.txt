# Глава 4. Списки



В системе «1С:Предприятие» для отображения информации чаще
всего используются таблицы, в которых выводятся списки объектов.



В зависимости от конфигурации и прав пользователя
список может быть доступен только для просмотра или также для редактирования:
могут создаваться новые элементы, изменяться или удаляться существующие записи.






Рис. 49.
Список документов «Продажи товаров»



Списки используются в двух типах
форм:



● 
Форма списка –
самостоятельное окно, предназначенное для работы со всеми элементами
справочника или документов: просмотр, создание, изменение, удаление,
перемещение, групповые операции, настройка данных, вывод списка. В форме списка
клавиша Enter
обычно открывает форму редактирования текущего элемента (поведение может быть
изменено на этапе конфигурирования).



● 
Форма выбора –
вспомогательное окно, которое открывается по кнопке выбора или клавише F4 при вводе
значения реквизита. Содержит упрощенный набор колонок и позволяет выбрать
элемент, создать новый (если разрешено), а также выполнить основные действия с
элементами.



При работе со
строками большинства действий удобнее выполнять через контекстное меню,
вызываемое правой кнопкой мыши. В контекстном меню доступны команды изменения,
копирования, удаления, навигации, поиска и другие.




## 4.1. Просмотр списка



Форма списка представляет собой таблицу, где каждая строка
соответствует элементу списка. Состав и порядок колонок определяются
конфигурацией. Над таблицей располагается командная панель с кнопками действий
и свернутой строкой поиска. В левой части таблицы могут отображаться служебные
значки и флажки для множественного выделения. При большом количестве данных
появляются полосы прокрутки.



Для перемещения по списку используются клавиши
стрелок, Page Up, Page Down,
Home и End или колесо
мыши. Щелчок по заголовку колонки выполняет сортировку по выбранному полю.






Рис. 50.
Форма списка «Товары»



Если количество строк не помещается в видимую область таблицы,
в нижней части списка отображается панель навигации. Панель содержит кнопки
перехода:



● 
В начало – к
первой странице списка;



● 
Предыдущая страница –
на предыдущий экран списка;



● 
Следующая страница –
на следующий экран списка;



● 
В конец – к
последней странице списка.



Панель расположена по центру под таблицей и отделена от нее
тонкой горизонтальной линией.



При переходе между страницами текущая строка не изменяется.



Если часть данных скрыта, верхняя граница видимой области
отмечается тонкой разделительной линией, а нижняя – эффектом рассеивания.



Для перемещения по списку можно использовать колесо мыши или
кнопки панели навигации.



Пользователь может изменить состав отображаемых колонок с
помощью команды Меню (⋮) – Изменить форму.



Методика настройки формы приведена в разделе «Настройка формы»,
здесь.



Ширина колонок сохраняется между сеансами. Если список
открыт в отдельном окне, сохраняются также размеры и положение окна.



Большинство действий над строками – выделение, переходы,
отбор по значению, открытие элемента, поиск – выполняются быстрее через
контекстное меню.




### 4.1.1. Иерархические списки



Платформа «1С:Предприятие» поддерживает иерархические списки
с несколькими уровнями вложенности. Максимальное число уровней задается в
конфигурации.



Возможны два варианта иерархии:



● 
иерархия групп и элементов – в списке есть
отдельные группы и элементы. Группа объединяет подчиненные группы и элементы,
элемент является конкретным объектом прикладного решения;



● 
иерархия элементов – любой элемент может
содержать подчиненные элементы, при этом набор реквизитов у всех элементов
одинаковый (например, список подразделений организации).




#### Режимы просмотра иерархических списков



Иерархический список может отображаться в одном из трех
режимов: Список, Иерархический
список, Дерево.



Режим просмотра выбирается командой Меню
(⋮) – Режим просмотра
и сохраняется между сеансами работы.



Режим «Список». В режиме Список в таблице отображаются все элементы и группы без
учета иерархии. Группы и элементы выводятся одной последовательностью; группы
помечены пиктограммой папки. Вложенность при этом не показывается, все строки
представлены как обычный плоский список.






Рис. 51.
Режим просмотра «Список»



Режим «Иерархический список».



В режиме Иерархический список в таблице
показываются элементы только текущего уровня. Путь к текущей группе
отображается в строке навигации над таблицей (например: Товары
> Электротовары > Пылесосы).






Рис. 52.
Список в режиме «Иерархический список»



● 
строка, соответствующая группе, содержит
пиктограмму папки и наименование группы;



● 
строка элемента списка представляет отдельный
объект (товар, контрагент и т. п.).



Чтобы открыть группу и перейти к ее содержимому, можно:



● 
дважды щелкнуть мышью по строке группы;



● 
выбрать группу и выполнить команду Меню (⋮) – Перейти
на уровень ниже;



● 
установить курсор на строке группы и нажать Ctrl + Стрелка вниз.



Для возврата к группе верхнего уровня можно:



● 
выбрать команду Меню (⋮) – Перейти на уровень выше;



● 
установить курсор в любой строке и нажать Ctrl + Стрелка вверх;



● 
воспользоваться ссылками в строке навигации
(выбрать нужный уровень в цепочке групп).



Режим «Дерево». В режиме Дерево все уровни иерархии отображаются в одной таблице.
Перед наименованием группы выводится значок раскрытия/сворачивания.






Рис. 53.
Иерархический список в режиме «Дерево»



● 
при нажатии на значок раскрытия рядом с группой
показываются подчиненные элементы;



● 
при нажатии на значок сворачивания подчиненные
элементы скрываются.



Для перемещения по дереву используются клавиши Стрелка вверх и Стрелка вниз.
Для одновременного раскрытия группы и перехода к подчиненному уровню можно использовать
Ctrl + Стрелка вниз, для возврата к
родительской группе – Ctrl + Стрелка вверх.



Для быстрого свертывания или разворачивания больших
фрагментов дерева можно использовать клавиши на цифровой клавиатуре:



● 
Shift + Alt + Num+
– развернуть выбранный узел и все подчиненные;



● 
Shift + Alt + Num–
– свернуть выбранный узел и все подчиненные;



● 
Ctrl + Shift + Alt + Num+
– развернуть все узлы дерева;



● 
Ctrl + Shift +Alt +Num–
– свернуть все узлы дерева.



Также в режимах Дерево и Иерархический список можно использовать клавиши Num+ и Num– для раскрытия и
сворачивания узлов дерева при установленном курсоре на соответствующей строке.



Для перехода в начало или в конец дерева используются
клавиши Home и End.



Списки с иерархией элементов. Для
списков с иерархией элементов режим Дерево
работает аналогично иерархии групп и элементов: элементы могут содержать
подчиненные элементы, при этом используются одинаковые реквизиты. В таких
списках визуальные пиктограммы строк, как правило, не различаются; структура
определяется отступами и значками раскрытия.






Рис. 54.
Список с иерархией элементов в режиме просмотра «Дерево»




## 4.2. Создание элемента списка



Чтобы создать новый элемент, используется команда Создать на командной панели формы списка.



Также можно нажать Insert, если
горячие клавиши не отключены конфигурацией.



Если список является иерархическим и требуется создать
элемент в определенной группе, сначала следует открыть нужную группу (в режиме Иерархический список или Дерево).
Новый элемент будет создан внутри этой группы.



Если форма элемента содержит реквизит Группа,
принадлежность можно изменить вручную – независимо от текущего уровня, в
котором был создан элемент.



Редактирование выполняется в форме элемента. Перед записью
может быть выполнена проверка корректности введенных данных. Чтобы сохранить
элемент, нужно нажать Записать и закрыть или аналогичную
команду. При закрытии формы через кнопку закрытия система запросит
подтверждение – Да для записи или Нет для отмены.



Для отказа от создания элемента используется клавиша Esc.




## 4.3. Создание группы списка



Если список поддерживает иерархию групп и элементов, в командной
панели формы списка доступна команда Создать группу.



Группа создается аналогично элементу списка – открывается
форма группы, содержащая реквизиты, заданные конфигурацией.



Чтобы новая группа сразу появилась внутри нужной родительской
группы, перед созданием следует открыть эту группу в режиме Иерархический список или Дерево.



Если форме группы доступен реквизит Группа,
можно выбрать любую родительскую группу, даже если элемент создавался в другом
разделе дерева.



Коды элементов и групп в иерархическом списке не должны
совпадать, если конфигурация использует уникальность кодов – система не
различает коды групп и элементов.




## 4.4. Копирование элемента списка



Для создания нового элемента по образцу используется команда Меню (⋮) – Скопировать или
команда Скопировать в контекстном меню строки
(быстрее).



После выполнения команды открывается форма нового элемента с
заполненными реквизитами. При отсутствии автонумерации код потребуется
изменить.




## 4.5. Редактирование элемента



Чтобы открыть элемент, можно:



● 
дважды щелкнуть по строке;



● 
нажать Enter;



● 
нажать F2;



● 
выбрать Изменить в
контекстном меню строки.



Поведение при открытии формы элемента зависит от выбранного
в интерфейсе режима работы окон:



● 
при режиме В закладках
форма элемента открывается в новой вкладке, расположенной рядом со вкладкой
списка;



● 
при режиме В диалоговых
формах форма элемента открывается в отдельном окне поверх списка, а
список остается доступным на фоне.



Конкретный способ зависит от настроек вида интерфейса,
выбранных пользователем.



После закрытия формы элемент возвращает управление списку;
изменения отображаются сразу.



Если редактирование недоступно (например, ограничено
правами), форма элемента открывается только для просмотра.



Команды редактирования и переходов доступны в контекстном
меню и выполняются быстрее, чем через Меню (⋮).




### 4.5.1. Перемещение элементов (реорганизация)



В иерархических списках элементы и группы можно переносить в
другие группы. Для этого используется команда:



● 
Меню (⋮) – Переместить в группу



● 
или команда Переместить в группу
в контекстном меню строки (удобнее).



Элементы также можно перетаскивать мышью, если конфигурация
поддерживает такое действие.



Нельзя переместить элемент в свою подчиненную группу.



Если используется уникальность кодов, система потребует
изменить код при совпадении.




### 4.5.2. Выбор значения из списка



Форма выбора открывается по кнопке выбора справа от поля
ввода или клавише F4. Она содержит сокращенный
набор колонок и предназначена для выбора элемента.



Enter в форме выбора выбирает
текущий элемент и закрывает окно.



Чтобы открыть элемент из формы выбора на редактирование,
используется команда F2 или команда Изменить в контекстном меню.



Если конфигурация разрешает, в форме выбора можно создать
новый элемент – через командную панель формы выбора.



Команды выбора, редактирования и создания доступны также
через контекстное меню строки.




## 4.6. Поиск



В форме списка доступна панель поиска,
расположенная в правой части командной панели. До начала ввода она отображается
как кнопка с иконкой поиска. Фокус в панель можно установить нажатием мышью или
просто начав ввод текста с клавиатуры – при вводе панель автоматически
превращается в строку поиска.






Рис. 55.
Панель поиска на форме списка



Поиск выполняется по всем видимым колонкам и обновляется по
мере ввода текста. Чтобы перейти к поиску вручную, можно нажать Ctrl+F.



Поиск также можно начать через контекстное меню строки.
Команда Найти: <Колонка> – <Значение>
выполняет поиск по выбранному значению конкретного поля, а команда Искать везде "<значение>" устанавливает
поиск по всем видимым колонкам. Когда поиск запускается из контекстного меню,
условие отображается под командной панелью в виде кнопки–фильтра. Такой фильтр
действует независимо от текста в строке поиска: при изменении текста список
уточняется, но установленное условие остается активным.



Чтобы отменить текстовый поиск, следует нажать кнопку
очистки справа в поле или клавишу Esc. Чтобы снять
условие, установленное через контекстное меню, нужно нажать крестик на кнопке–фильтре
под командной панелью. Удаление текста из строки поиска не отменяет параметр,
заданный кнопкой–фильтром.



Поиск работает по вхождению текста в значения видимых
колонок. Найденные фрагменты подсвечиваются. Если часть колонок скрыта, поиск
выполняется только по отображаемым данным. Текстовый поиск и кнопки–фильтры
работают совместно: фильтр задает исходный набор строк, а текст в строке поиска
уточняет результат.




## 4.7. Настройка списка



Настройка отображения данных, отбора, сортировки,
группировки и условного оформления выполняется в диалоге Настройки
списка.



Диалог открывается из формы списка командой



Меню (⋮) – Настройки списка – Настроить список.






Рис. 56.
Форма настройки списка



В диалоге можно настроить:



● 
отбор данных, отображаемых в списке;



● 
поля и порядок сортировки строк;



● 
условное оформление данных;



● 
группировку записей;



● 
другие параметры, возможность изменения которых
предусмотрена разработчиком.






Все параметры распределены по вкладкам: Основные, Фильтр, Сортировка, Условное оформление
и Группировка.



Настройка применяется к списку, если для соответствующего
элемента включен флажок использования. Для элементов одной настройки можно
задать порядок применения.



Настройка отображения данных в списке выполняется по тем же
принципам, что и настройка варианта отчета в системе компоновки данных.



Чтобы вернуть настройки списка к значениям по умолчанию,
выполните команду Меню (⋮)
–
Настройки
списка
–
Установить
стандартные
настройки.



Быстрый отбор по значению доступен через контекстное меню
строки списка.



Настройки сохраняются между сеансами работы.




### 4.7.1. Настройка состава колонок



Состав, порядок следования и видимость колонок настраиваются
отдельно – через меню формы списка.



Чтобы изменить состав колонок, выполните команду Меню (⋮) – Изменить форму и
укажите элементы, которые должны отображаться в форме списка.



Настройки колонок не относятся к настройкам списка.
Настройки списка используются для управления отображением данных (отбор,
сортировка, группировка, условное оформление), но не для изменения структуры
формы.




### 4.7.2. Фильтр



Чтобы задать отбор данных в списке, откройте
диалог настройки списка и перейдите на вкладку Фильтр.






Рис. 57.
Настройка фильтра в списке.



Для добавления условия нажмите
кнопку Добавить фильтр и выберите поле, по
которому будет выполняться отбор. Для каждого условия указывается вид сравнения
и значение. Значение можно выбрать из списка или ввести вручную, если поле это
допускает.



Если нужно добавить несколько условий
сразу, используйте команду Добавить несколько. Для
объединения условий в блоки используйте команду Добавить
группу.



Порядок элементов в списке
определяет последовательность их применения при формировании отчета. Порядок
элементов фильтра можно изменить с помощью кнопок перемещения вверх и вниз, расположенных
в правой части строки элемента.



Использование условия включается
или отключается с помощью переключателя в строке условия. Чтобы быстро включить
или отключить все условия фильтра, нажмите Меню (⋮) и выберите команду Включить все или Выключить все.



Если требуется объединить
несколько условий, выделите их в списке. На командной панели появится команда Сгруппировать, которая позволяет создать группу из
выбранных условий. При необходимости выбранные условия можно удалить командой Удалить.



После изменения условий нажмите Применить
в форме настроек.



Подробнее о работе с фильтром и о видах сравнения см. раздел
«» на стр.







### 4.7.3. Сортировка



Чтобы настроить сортировку строк списка, откройте
диалог настройки списка и перейдите на вкладку Сортировка.






Рис. 58.
Настройка сортировки списка



Для добавления элемента сортировки нажмите Добавить сортировку.



Для каждого элемента укажите направление сортировки: По возрастанию или По убыванию.



Если задано несколько полей сортировки, строки списка
сортируются последовательно в порядке, установленном для элементов сортировки.






Порядок применения элементов сортировки можно изменить с помощью
команд перемещения.



Сортировка применяется, если для элемента включен флажок
использования.



Чтобы применить настройки сортировки, нажмите Применить.



Подробнее о работе с сортировкой см. раздел «Сортировка»,
здесь.





Примечание. Сортировка по полям,
содержащим строки неограниченной длины, не поддерживается.







### 4.7.4. Условное оформление



Для настройки условного оформления откройте
диалог настройки списка и перейдите на вкладку Условное
оформление.






Рис. 59.
Настройка условного оформления в списке



Чтобы добавить элемент условного оформления, нажмите Добавить.



Редактирование элемента выполняется в отдельном окне, в
котором задаются:



● 
На закладке Оформление –
параметры оформления, такие как цвет фона, цвет текста и другие визуальные
характеристики;



● 
На закладке Условие –
условие применения оформления, задаваемое аналогично условиям отбора;



● 
На закладке Поля –поля
списка, к которым будет применяться оформление при выполнении условия.



Если задано несколько элементов условного оформления, они
применяются последовательно в порядке, указанном в списке.



Если для одного элемента списка выполняются условия
нескольких оформлений, применяется последнее из них.



Подробнее о настройке условного оформления см. раздел «Условное оформление»,
здесь.



Чтобы применить настройки условного оформления, нажмите Применить, и затем Применить.




### 4.7.5. Группировка



Чтобы сгруппировать записи в списке, откройте
диалог настройки списка и перейдите на вкладку Группировка.






Рис. 60.
Настройка группировки в списке



Для добавления группировки нажмите Добавить
группировку.



Записи в списке группируются по выбранному полю.



Если задано несколько полей группировки, данные группируются
последовательно в порядке, установленном для элементов группировки.



Порядок применения элементов группировки можно изменить с
помощью команд перемещения.



Группировка применяется, если для элемента включен флажок
использования.



Чтобы применить настройки группировки, нажмите Применить.





Примечание. Группировка списка по
полям, содержащим строки неограниченной длины, не поддерживается.







### 4.7.6. Период



Некоторые списки поддерживают фильтрацию по
периоду. Настройка периода выполняется командой: Меню
поиска (⋮) – Установить период.






Рис. 61.
Меню поиска списка



Период можно выбрать из стандартных вариантов или указать
вручную.



Если установлен флажок Запомнить
выбранный период, выбранное значение будет использоваться при следующем
открытии списка.






Рис. 62.
Настройка периода в списке



Для некоторых конфигураций команды работы с периодом могут
быть доступны и через контекстное меню.
