# Глава 3. Работа в формах



В «1С:Предприятии» пользователь работает с формами. Форма
может содержать различные элементы, с помощью которых информация отображается и
может быть изменена пользователем: поля ввода, командные панели, кнопки, флажки,
закладки, таблицы, различные диаграммы, поля текстового, табличного и HTML–документов, поля
географической и графической схем. Данная глава содержит описание общих
принципов работы с формами и элементами форм.




## 3.1. Общие принципы работы с формами



При работе в интерфейсе Версия 8.5
действия в формах выполняются, как правило, однократным нажатием
левой кнопки мыши.



Двойной щелчок используется для
открытия формы объекта из списка (например, элемента справочника или
документа).



Для остальных действий применяются команды панели формы,
контекстное меню или кнопки формы.



Для перемещения по элементам формы используются клавиши Tab и Shift + Tab или
щелчок мыши по нужному элементу. Некоторые элементы, такие как командные
панели, могут быть недоступны для перехода по Tab.
В этих случаях используется мышь или сочетание Alt + F10.



При вводе данных можно переходить к следующему элементу
нажатием клавиши Enter. Последовательность
переходов определяется разработчиком конфигурации. Нажатие Enter на строке списка также открывает форму объекта,
аналогично двойному щелчку.




## 3.2. Поле ввода



Поле ввода предназначено для просмотра, ввода и
редактирования данных различных типов. Внешний вид и набор доступных действий
зависят от типа данных и настроек формы.



Чтобы ввести значение, нажмите левую кнопку мыши в области
поля или перейдите к нему клавишей Tab.



Доступны стандартные операции редактирования:
ввод, удаление, копирование, вставка из буфера обмена и отмена последних
действий.






Рис. 15.
Пример формы с полями ввода



Поля, обязательные для заполнения, отмечаются красной
звездочкой перед наименованием.



Поля, доступные только для просмотра, отображаются с серым
фоном.



Если значение не введено, система не позволит сохранить
форму до заполнения всех обязательных полей.




### 3.2.1. Элементы управления



В поле ввода могут
располагаться дополнительные элементы:



● 
кнопка очистки – удаляет
введенное значение;



● 
кнопка выбора –
открывает форму выбора значений;



● 
кнопка выпадающего списка
˅ – отображает варианты выбора;



● 
кнопки регулирования (+
/ – или ˄ / ˅) – изменяют числовое или порядковое значение;



● 
кнопка замка –
указывает, что поле недоступно для редактирования;



● 
кнопка дополнительных действий
(⋮) – открывает контекстное меню.



Кнопки очистки и выбора находятся справа от поля.



Элементы регулирования располагаются снаружи поля – с одной
или обеих сторон, в зависимости от выравнивания текста.



Иконка поля (если задана) отображается слева и остается
видимой всегда – при пустом и заполненном значении.




### 3.2.2. Подсказка внутри поля



Если для поля задан текст подсказки, он отображается внутри
поля светло–серым шрифтом, пока поле пустое.



Подсказка помогает пользователю понять, какие данные
требуется ввести.



После заполнения поля подсказка исчезает.




### 3.2.3. Картинка в поле ввода



В левой части поля может отображаться иконка, поясняющая
содержание вводимых данных, например телефон, адрес, поиск или календарь.



Картинка отображается постоянно – и при пустом, и
при заполненном поле.






Рис. 16.
Пример картинки в поле ввода



Для полей выбора цвета рядом с текстом отображается
образец выбранного цвета.






Рис. 17.
Пример поля выбора цвета



Цвет предварительного просмотра не зависит от темы
оформления приложения.




### 3.2.4. Выпадающий список и поиск значения



Если для поля предусмотрен выбор из списка, при вводе текста
выполняется поиск подходящих значений. Если совпадения не найдены, в верхней
части списка отображается сообщение Не найдено, а
ниже показываются возможные похожие варианты.






Рис. 18.
Выпадающий список в поле ввода



При наличии большого числа элементов или при использовании
кнопки создания в выпадающем списке появляется строка поиска. В нее
автоматически подставляется текущий текст поля ввода, а если значение уже
выбрано – оно отображается в строке поиска при открытии списка.



В полях ввода типа Строка и Число при вводе текста может появляться элемент Ввести, если такого значения нет в списке. В полях с
множественным выбором вместо него отображается Добавить.
Элемент не появляется, если введенное значение полностью совпадает с
существующим элементом списка или не соответствует типу поля.



При выборе элемента Ввести
введенное значение применяется в поле ввода, а список закрывается. Добавленные
пользователем значения отображаются при следующем открытии списка: для
одиночного выбора сохраняется только последнее пользовательское значение, для
множественного – сохраняются все выбранные значения. Если по введенному тексту
невозможно подобрать элементы и отображается состояние Не
найдено, после закрытия списка в поле ввода остается введенное значение.



Выбор и навигация. Чтобы открыть
список, используйте клавишу F4. Чтобы открыть
форму выбранного элемента, нажмите Ctrl + Shift + F4.
Навигация по списку выполняется стрелками; наведенный элемент подсвечивается.
Элементы, совпадающие со значением поля ввода, получают активное состояние.



Закрытие выпадающего списка.
Выпадающий список закрывается при выборе значения, при нажатии Esc, при щелчке вне области списка или при изменении
размеров окна.




### 3.2.5. Кнопка выбора



Кнопка выбора используется для открытия формы, из
которой выбирается значение, например элемент справочника или документ.






Рис. 19.
Кнопка выбора в поле ввода



Если рядом с полем отображается и кнопка выбора, и кнопка
списка, система показывает только одну из них – в зависимости от контекста,
чтобы избежать дублирования действий.



В нижней части списка может отображаться кнопка с надписью,
например Все поставщики или Показать
все. Текст поясняет, какие данные будут открыты при выборе в форме.



Эта же надпись выводится во всплывающей подсказке у кнопки
выбора.




### 3.2.6. Кнопка очистки



Кнопка очистки появляется в правой части поля,
если в нем введено значение и редактирование разрешено.






Рис. 20.
Кнопка очистки в поле ввода



В мобильной версии кнопка очистки отображается всегда.



Если редактирование запрещено, кнопка и команда Очистить в контекстном меню недоступны.



Выполнить очистку также можно сочетанием Shift + F4.




### 3.2.7. Кнопка замка



Если
редактирование значения ограничено, поле отображается с серым фоном и кнопкой
замка справа.






Рис. 21.
Защищенное поле ввода



Чтобы временно разрешить изменение, нажмите кнопку замка или
клавишу F2.



Повторное нажатие возвращает ограничение.




### 3.2.8. Поле для ввода пароля



В некоторых случаях поле ввода используется для ввода пароля
или другого конфиденциального значения.



При этом введенные символы заменяются точками.



Поле не отображает реальный текст, чтобы защитить
данные от просмотра другими пользователями.

Контекстное

В
меню размещаются команды, связанные с редактированием и дополнительными
действиями.



Состав
команд задается в конфигурации.






Рис. 23. Пример контекстного меню поля




### 3.2.10. Добавление нескольких строк



В некоторых полях можно вводить текст в несколько строк.



Такой способ используется, когда требуется короткое
текстовое пояснение или заметка.



Чтобы перейти на новую строку, нажмите Enter.






Рис. 24.
Пример многострочного поля ввода



Многострочный ввод доступен только для полей типа Строка, если для них не включен режим пароля и режим
выбора из списка.



В многострочных полях не отображаются кнопки внутри поля
(выбора, очистки, действий и т. д.). При необходимости используйте команды
контекстного меню.




### 3.2.11. Выбор даты и времени



При вводе даты и времени можно выбрать значение
из календаря.






Рис. 25.
Ввод даты в поле ввода с помощью календаря



Панель выбора времени появляется при наличии времени в
составе даты и содержит барабаны часов и минут с шагом от 1 до 60 минут.



В строке Выбрано… отображается
выбранное значение, которое будет записано в поле после нажатия Подтвердить.



Если шаг выбора времени составляет 10 минут и более, панель
отображается в виде табличной сетки, где активной становится ячейка с ближайшим
меньшим временем.



Если время не требуется, панель не отображается.




## 3.3. Флажок



Флажок предназначен для включения или выключения параметра.



Значение флажка может быть установлено (включено)
или снято (выключено).






Рис. 26.
Пример флажка на форме



Чтобы изменить состояние флажка, нажмите левую кнопку мыши
или клавишу Пробел, когда флажок активен.



При этом напротив наименования параметра отображается
галочка. Повторное нажатие снимает ее. В установленном состоянии рядом с
наименованием отображается галочка. Повторное нажатие снимает отметку.



Флажки применяются для задания логических параметров, выбора
режимов, включения дополнительных опций и фильтров.



Если флажок недоступен для изменения, он отображается в
ослабленном виде.



При установке ограничений редактирования изменение состояния
флажка невозможно.



При автоматическом определении состояния отображается только
текущее значение без возможности изменения пользователем.



Флажки могут располагаться отдельно или в составе групп
элементов формы.



Если группа содержит несколько флажков, они обычно
используются для независимого выбора нескольких параметров.




### 3.3.1. Вид «Выключатель»



Некоторые флажки могут отображаться в виде
выключателя.



В этом варианте элемент представляет собой
компактный горизонтальный переключатель с бегунком.






Рис. 27. Выключатель



Включенное состояние выделяется цветом
акцента интерфейса.



Выключенное состояние отображается светлым
тоном.



По действию выключатель полностью соответствует
флажку:



его можно переключать щелчком мыши или
клавишей Пробел.



Такой вид используется там, где важно
подчеркнуть текущее состояние параметра.




## 3.4. Тумблер



Тумблер предназначен для переключения значения параметра между
двумя состояниями – включено и выключено.



В отличие от флажка, тумблер используется для
более наглядного управления режимами или настройками, когда важно визуально
показать текущее состояние параметра.






Рис. 28.
Тумблер состояния заказа



Включенное состояние отображается цветом акцента, заданным
для интерфейса, а выключенное – серым.



Если подписи не определены, отображается только сам
переключатель.



Чтобы переключить тумблер, нажмите левую кнопку мыши или
клавишу Пробел.



Состояние изменяется сразу, без подтверждения.



Тумблеры применяются для управления настройками, включения
дополнительных функций и переключения режимов.



Если элемент недоступен, он отображается в неактивном виде и
не реагирует на нажатия.



В некоторых формах состояние тумблера может изменяться
автоматически при изменении других параметров формы.



Если значение тумблера изменяется программно, элемент
обновляется сразу – перезапуск формы не требуется.



Если элемент недоступен для редактирования, он отображается
в неактивном (ослабленном) виде и не реагирует на нажатия.

клавишами управления курсором выбрать требуемое значение и нажать клавишу Enter.

текущего состояния реквизита формы, с которым он связан (определяется при

## 3.7. Полоса регулирования



Полоса регулирования предназначена для быстрого и
наглядного ввода числовых значений. По внешнему виду он напоминает обычный
линейный регулятор (на рисунке под полем ввода, с которым он связан),
расположенный вертикально или горизонтально.






Рис. 31.
Регулятор значения



Значение реквизита, связанного с этим элементом,
определяется положением бегунка и значениями минимума и максимума,
установленными при проектировании конфигурации.



Положение бегунка может быть изменено клавишами управления
курсором, но с помощью мыши удобнее.




## 3.8. Гиперссылка



В форме могут
располагаться элементы управления, представляющие собой текст, при наведении
указателя мыши на который указатель принимает вид руки и появляется
подчеркивание. Такой элемент называется Гиперссылка. Он предназначен для перехода к некоторому
объекту (например, открытия других форм).






Рис. 32. Гиперссылка на форме



Для выбора
гиперссылки следует щелкнуть по ней мышью или сделать ее активной клавишами Tab и Shift + Tab
(или клавишами Стрелка вправо, Стрелка влево, Стрелка вниз, Стрелка вверх) и нажать клавишу Enter.
Настройка отработки нажатия гиперссылки определяется при проектировании
конфигурации. Гиперссылку могут иметь и картинки, размещенные в форме. При
поднесении указателя мыши к картинке с гиперссылкой форма указателя также
принимает вид, как на рисунке ниже.




## 3.9. Кнопка



Кнопки используются для выполнения действий, доступных на
форме. Они могут содержать заголовок, картинку или оба элемента. Внешний вид
кнопки зависит от настроек конфигурации: важности кнопки, наличия фонового
оформления и того, отображается ли кнопка в виде карточки.



Кнопку можно выбрать мышью или сделать активной с
помощью клавиш Tab и Shift+Tab,
после чего нажать Enter.






Рис. 33.
Кнопка на форме



Если для кнопки предусмотрено сочетание клавиш,
действие будет выполнено при нажатии этой комбинации. Недоступные кнопки
отображаются в неактивном состоянии.






Рис. 34.
Недоступная кнопка




### 3.9.1. Важность кнопок



Кнопки могут иметь разную важность. Важность определяет,
насколько заметным будет действие на форме. Оформление кнопки зависит от
конфигурации, поэтому разные кнопки могут выглядеть по–разному.



Главная кнопка выделяет
основное действие. Она заметна визуально и используется в ключевых сценариях
работы.






Рис. 35.
Главная кнопка



Обычная кнопка применяется для большинства действий и
отображается стандартно.



Дополнительная кнопка
используется для второстепенных действий. Она отображается без собственного
фона, поэтому выглядит легче остальных элементов.






Рис. 36.
Дополнительная кнопка




## 3.10. Карточка



Карточка используется для компактного представления
связанных данных. Она объединяет несколько элементов в одну сущность:
изображение, заголовок и дополнительные части формы. Карточка делает информацию
наглядной и упрощает переход к связанным сведениям.



Карточка обычно содержит небольшое количество данных и
служит точкой входа к более детальной информации. Такой элемент помогает
структурировать форму и делает интерфейс понятным.



Карточка может отображать изображение, текст и
элементы управления. Например, карточка товара может содержать наименование,
картинку и команду, связанную с действием.






Рис. 37.
Карточка товара



Карточка может иметь фоновый цвет или фоновое
изображение. Фон может отображаться с полупрозрачностью или с эффектом
размытия.






Рис. 38.
Карточка с фоном



При необходимости карточка может выполнять роль гиперссылки.
В этом случае нажатие на любую область карточки открывает подробную информацию
или выполняет действие, определенное конфигурацией. Такой вариант позволяет
быстро переходить к связанным данным.



Карточка может использоваться и как элемент управления.
В этом случае она оформляется как визуальная команда: изображение и заголовок
объединены в один блок, нажатие на который выполняет действие.






Рис. 39.
Карточка команды



Размещение изображения и текста в карточке может отличаться.
Изображение может находиться сверху или снизу. Конкретное размещение зависит от
конфигурации.




## 3.11. Таблица



Таблица используется в форме для отображения
данных в нескольких колонках. Каждая строка таблицы содержит набор значений,
связанных с реквизитами. Состав колонок и доступные действия определяются в
прикладном решении.






Рис. 40.
Таблица со списком товаров в документе «Продажа»



Таблица может использоваться только для просмотра данных или
быть доступной для редактирования.



Если данных много, используется вертикальная прокрутка. Если
значения не помещаются по ширине, доступна горизонтальная прокрутка.



Таблица может отображать строки в несколько уровней, когда в
одной строке выводится расширенный набор данных.



Если значение не помещается в ячейку, показывается первая
строка текста с многоточием.



Выбор строк. Строку можно выбрать мышью или клавишами
управления курсором. Если форма позволяет множественный выбор, удерживайте
клавишу Ctrl.



Выбор строк с помощью флажков. В
таблицах интерфейса Версия 8.5 строки можно
выбирать не только щелчком мыши, но и с помощью флажков в первой колонке.
Флажок в строке позволяет отметить одну или несколько строк для выполнения
действий, доступных в форме.



В заголовке первой колонки расположен флажок Выделить все. При установке этого флажка выбираются все
строки таблицы. Если снять флажок, выделение всех строк будет снято. Когда
отмечена только часть строк, флажок Выделить все
отображается в промежуточном состоянии.



Такой способ удобен для массового выбора строк, если форма
поддерживает выполнение действий над несколькими строками сразу.



Просмотр значений. При наведении
указателя на ячейку отображается подсказка с полным текстом.



Настройка колонок. Ширина колонок
изменяется перетаскиванием границы колонки.При удержании клавиши Ctrl изменяется только текущая колонка.



Если форма позволяет настраивать порядок или видимость
колонок, это выполняется через Меню таблицы (⋮).




### 3.11.1. Команды работы с таблицей



Таблица имеет собственный набор команд, относящихся только к
данным внутри нее. Они доступны в двух местах:



● 
Контекстное меню таблицы –
Открывается по щелчку правой кнопкой мыши в области таблицы или на
ячейке.



● 
Меню таблицы (⋮) –
Если таблица содержит собственную кнопку Меню (⋮), она располагается в области элемента
таблицы и управляет только действиями таблицы.



Команды таблицы включают:



● 
сортировку;



● 
поиск;



● 
копирование строки;



● 
удаление строки (если разрешено);



● 
действия по выделенной строке;



● 
действия по ячейке;



● 
дополнительные команды, связанные с
отображаемыми данными.




### 3.11.2. Контекстное меню таблицы



Контекстное меню открывается по правой кнопке мыши в области
таблицы или на ячейке. Состав команд зависит от текущего состояния
таблицы – есть ли выделенные строки или текущая строка.



Если включено автозаполнение контекстного меню, в него
автоматически добавляются команды, применимые к таблице или строкам:



● 
при наличии текущей строки –
системные команды, команды объекта метаданных и общие команды, связанные с
типом таблицы;



● 
при множественном выделении
строк – команды, применимые к нескольким строкам, включая действия
командной панели, поддерживающие множественное выделение;



● 
если таблица пустая –
только системные и вручную добавленные команды.



Команды с большей важностью отображаются выше, а дубли
команд автоматически исключаются – в меню остается первая из них.



Контекстное меню можно открыть не только мышью, но и
клавишей открытия контекстного меню (обычно Меню (⋮) или Shift + F10).
Если активна ячейка, меню откроется в ее левом верхнем углу, иначе – в
левом верхнем углу таблицы.



В режиме множественного выделения строк меню также доступно:



● 
при щелчке правой кнопкой по выделенной строке
выделение не меняется;



● 
при щелчке по невыделенной строке предыдущее
выделение сбрасывается и активизируется новая строка.




### 3.11.3. Редактирование данных



Начало редактирования. Чтобы
изменить значение, выберите ячейку и начните ввод или нажмите Enter.



Если доступен список значений, нажмите
Ctrl + Стрелка вниз.



Переход между ячейками.



● 
Enter или Tab – переход к следующей редактируемой ячейке.



● 
Shift + F2 –
завершение редактирования строки.



Добавление строки. Нажмите Insert. Новая строка автоматически доступна для ввода.



Если форма поддерживает автосоздание строки, новая строка
появляется при нажатии Стрелка вниз в последней
строке.



Если строка не заполнена, она удаляется автоматически.



Удаление строки. Выберите строку и
нажмите Delete.





Внимание! Удаление выполняется без
предупреждения.






Копирование строки. Выберите строку
и нажмите F9. Будет создана строка со значениями
исходной.

Нумерация строк. Если таблица

Сортировка. Для сортировки
используйте команду из контекстного меню или Меню

Если форма допускает свободное упорядочивание, строки можно

Навигация



● 
PgUp и PgDwn – постраничное перемещение,



● 
Ctrl + Hme –
первая строка,



● 
Ctrl + End –
последняя строка.



Поиск. В контекстном меню доступна
команда Искать везде.



Чтобы перенести значение ячейки в поиск, нажмите Alt + F.




## 3.12. Группы



Группы используются для объединения элементов формы и
удобного размещения данных. С помощью групп можно структурировать поля, колонки
таблиц, команды и другие элементы, а также управлять их отображением.



Группа определяет, как элементы распределяются
внутри формы: в одном столбце, в нескольких столбцах, в виде отдельного блока
или в составе страницы.






Рис. 41.
Обычная группа полей на форме



Группа отображается на форме как визуально объединенный блок
элементов. Границы группы могут быть выделены отступами, рамкой или
разделительной линией.



Группы могут содержать один или несколько элементов формы:
поля, таблицы, команды, переключатели, флажки и другие вложенные группы. Группы
могут иметь заголовок и особое оформление, упрощающее восприятие информации.
Например, группа полей может быть визуально отделена рамкой или линией.



Группы помогают организовать элементы формы и делают
структуру данных более наглядной. Внутри группы могут находиться вложенные
группы, поля, таблицы и другие элементы, которые отображаются как единый блок.
В зависимости от задачи прикладного решения группа может быть обычной, свертываемой
или всплывающей, при этом способ отображения выбирается разработчиком и может
различаться в разных формах. Пользователь не изменяет тип групп и не перемещает
элементы между ними – такие действия выполняются только при разработке формы.



Свертываемые группы. Свертываемая
группа позволяет временно скрыть свое содержимое. В развернутом состоянии
показываются все элементы группы, а в свернутом – только ее заголовок и значок
сворачивания.



Чтобы изменить состояние группы, нажмите на ее
заголовок.






Рис. 42.
Свертываемая группа



Свертываемые группы особенно удобны на формах с большим
количеством данных, когда часть информации используется реже и ее можно скрыть,
не нарушая общую структуру формы.



Всплывающие группы. Всплывающая
группа отображается на форме в виде заголовка. При нажатии на заголовок
содержимое открывается во всплывающем окне поверх формы. После закрытия окна
пользователю снова доступна исходная форма.






Рис. 43.
Всплывающая группа



Такой способ отображения используется для данных, которые
занимают много места или нужны не постоянно. Если у группы не отображается
заголовок, она ведет себя как обычная группа и не открывается во всплывающем
окне.




### 3.12.1. Командная панель



Форма может содержать группы элементов вида Командная панель. Каждая командная панель представляет
собой набор кнопок (команд), действие которых распространяется только на данную
форму или на отдельную часть формы.



Для активизации командной панели можно использовать
комбинацию клавиш Alt + F10. При работе
с клавиатуры командная панель участвует в обходе элементов формы по клавише Tab или Shift + Tab.
Обходятся все кнопки, а при нажатии кнопки панели (клавишей Enter) выполняется действие.




### 3.12.2. Страницы



Страницы используются для размещения элементов
формы в виде набора закладок. Такой способ позволяет объединить данные по
смыслу и упростить навигацию внутри формы. В интерфейсе страницы отображаются
как горизонтальная панель закладок: каждая закладка имеет собственный
заголовок, а активная закладка выделена визуально.






Рис. 44.
Страницы в документе «Продажа»



Чтобы открыть нужную часть формы, достаточно нажать на
заголовок закладки. Между закладками можно переключаться также сочетаниями
клавиш Ctrl + PgUp и Ctrl +
PgDn. После выбора отображается содержимое выбранной вкладки – поля,
группы, таблицы, команды и другие элементы формы. Если данных много, область
страницы можно прокручивать. Переключение между закладками не влияет на уже введенные
значения: данные сохраняются автоматически.



В формах могут встречаться вложенные страницы –
дополнительные закладки внутри области страницы. Они работают так же, как
основные: переключение выполняется нажатием на заголовок, а содержимое отображается
в пределах выбранной вкладки.



Когда заголовки страниц не помещаются по ширине, в панели
закладок появляется пункт Еще. В нем отображаются
закладки, скрытые из–за нехватки места, и через него можно перейти к любой из
них.



Страницы помогают структурировать большие формы и позволяют
скрывать реже используемые данные, чтобы не перегружать рабочую область.
Порядок закладок, их названия и состав определяются прикладным решением и
недоступны для изменения пользователем. В обоих вариантах открытия окон – в
закладках и в диалоговых окнах – страницы работают одинаково.




## 3.13. Диаграмма



Диаграмма может быть встроена в форму, табличный документ
или использоваться в составе отчета.



В интерфейсе Версия 8.5 внешний
вид диаграммы приведен в соответствие с актуальными стилями платформы.
Поддерживается отображение в светлой и темной темах: при смене темы диаграмма
автоматически адаптирует цвета и стиль оформления.



В этом интерфейсе:



● 
используется палитра по умолчанию – Насыщенная;



● 
для текста и фона применяется автоматический
выбор цвета;



● 
расположение легенды и заголовка определяется
автоматически с учетом отступов;



● 
при добавлении диаграммы на форму не задаются
свойства рамки и фона – используются значения по умолчанию.



Если диаграмма встроена в форму, управление
осуществляется через элементы формы, предусмотренные прикладным решением. При
наличии соответствующих настроек может быть доступна смена типа диаграммы.






Рис. 45.
Пример отображения диаграммы в темной теме



О работе с диаграммами в составе табличного документа см.
приложение «Редактор
табличных документов».




## 3.14. Поле картинки



Элемент Поле
картинки предназначен для размещения в форме различных картинок.
Этот элемент также может содержать ссылку (см. описание в разделе «Гиперссылка»).



Для элемента формы Поле картинки добавляется контекстное меню,
состоящее из пунктов Копировать, Сохранить как, Просмотреть картинку.



Чтобы скопировать изображение в буфер обмена, нужно выбрать
пункт меню Копировать.
После этого его можно вставить в другое приложение.



Меню Сохранить
как доступно для картинок, загруженных из файла. С его помощью можно
сохранить картинку в файл на диске.



Если выбран пункт Просмотреть картинку, то на экран выводится окно
просмотра картинки, в котором есть кнопки увеличения и уменьшения масштаба, а
также установки реального размера. Существует возможность изменять размеры
окна. Для выбора нужного масштаба используются пункты меню Увеличить, Уменьшить и Оригинальный размер. Для
изменения масштаба можно также использовать клавиши, расположенные на цифровой
клавиатуре. Нажатие клавиши + (плюс)
увеличивает картинку, клавиши – (минус) –
уменьшает. Команда Оригинальный
размер устанавливает реальный размер картинки.



Изменение масштаба также можно выполнять с клавиатуры,
используя клавиши + (плюс)
и – (минус), расположенные
справа от цифровой клавиатуры.




## 3.15. Поле текстового документа



Поле текстового документа
предназначено для работы с текстами. В нем размещается окно текстового
редактора, предназначенное для ввода и сохранения строк текста. Подробнее о
работе с редактором см. приложение «Редактор текстового документа».




## 3.16. Поле табличного документа



Форма может содержать встроенный табличный документ (элемент
формы Поле табличного документа). О работе с
элементом формы Поле табличного документа см. в приложении
3, поставляемом в электронной версии документации.



Перемещение по таблице. Окно
табличного документа в любой момент времени показывает только часть таблицы,
расположенной в этом окне. Для выбора ячейки таблицы используется мышь.
Выбранная ячейка становится активной. Для вывода на экран областей, не
поместившихся в границах окна, используются линейки прокрутки.





Совет. Чтобы прокрутить документ
по горизонтали, нужно нажать и удерживать клавишу Shift,
прокрутить колесо мыши.






Для удобства прокрутки большого документа можно нажать колесо
мыши и выбрать курсором направление прокрутки. Скорость прокрутки зависит от
расстояния между текущим положением курсора и тем местом, где было нажато
колесо (отображается специальным образом).



В контекстном меню ячейки доступна команда Искать везде, при выборе которой выполняется глобальный
поиск данных по значению текущей ячейки. Подробнее см. раздел «Глобальный поиск»
здесь.





Совет. Чтобы скопировать значение
ячейки в поле глобального поиска можно использовать сочетание клавиш Alt + F.






Управление видом таблицы. При помощи
пунктов подменю Меню (⋮)
– Таблица – Вид можно управлять
отображением различных областей и компонентов табличного документа: заголовков строк
и колонок, сеткой таблицы и т. д.



Кратко поясним назначение пунктов меню Вид. Каждый пункт работает как переключатель: выбор (нажатие)
вызывает или прекращает отображение соответствующей области.







Пункт меню




Действие








Зафиксировать таблицу




Фиксирует верхние строки и левые колонки табличного
документа таким образом, чтобы при перелистывании табличного документа они
постоянно присутствовали на экране







Отображать сетку




Включает и отключает показ сетки табличного
документа – тонких линий, разделяющих ячейки таблицы







Отображать заголовки




Управляет показом заголовков строк и колонок табличного
документа







Отображать группы




Включает и отключает показ групп – областей слева от
заголовков строк и сверху от заголовков колонок







Редактирование




Включает и отключает запрет на любые изменения в
табличном документе







Отображать примечания




Включает и отключает режим показа примечаний

Отключает и включает показ цветов, заданных для

Режим просмотра страниц




Отключает и включает показ минимального количества строк
(определяется введенной информацией), области печати, разрывов страниц







Масштаб




Дискретно масштабирует изображение. При выборе этого
пункта открывается подменю, в котором можно выбрать желаемый масштаб
изображения табличного документа









Совет. Для
масштабирования можно использовать мышь с колесом прокрутки. Нужно нажать
клавишу Ctrl и, не отпуская ее, повернуть колесо.
Движением колеса от себя масштаб увеличивается, к себе – уменьшается.






Фиксация таблицы (шапки и боковой части).
Редактор табличных документов системы «1С:Предприятие» позволяет зафиксировать
верхние строки и левые колонки табличного документа таким образом, чтобы при
прокрутке табличного документа они постоянно присутствовали на экране.



Чтобы зафиксировать верхние строки табличного документа,
следует выделить строку табличного документа целиком и выбрать пункт Меню (⋮) –Таблица –
Вид – Зафиксировать таблицу. Зафиксированы будут все строки,
расположенные выше выделенной строки.



Чтобы зафиксировать колонки табличного документа, следует
выделить колонку табличного документа целиком. Зафиксированы будут колонки,
расположенные слева от выделенной колонки.



Чтобы зафиксировать одновременно строки и колонки табличного
документа, следует выделить одну ячейку табличного документа. Зафиксированы
будут строки и колонки, расположенные соответственно выше и левее выделенной
ячейки.



Для отмены фиксации следует повторно выбрать пункт Зафиксировать таблицу.



Защита от редактирования. Для
запрета редактирования табличного документа используется пункт Меню (⋮) –Таблица –
Вид – Редактирование, который должен быть отключен.



Если ячейка содержит текст, который полностью не помещается
в видимой области, то при подведении указателя мыши текст показывается в виде
подсказки. Если размер текста достаточно большой, чтобы отобразить его в виде
подсказки, показывается только часть текста.




## 3.17. Поле форматированного документа



На форме может быть расположен
элемент вида Поле форматированного документа,
предназначенный для хранения текстового документа, включающего элементы
форматирования: выделение цветом, изменение шрифта текста, гиперссылки,
картинки и т. д.



Такое поле может использоваться,
например, при создании сообщения электронной почты или для редактирования
простейшего html–документа.



Если поле форматированного
документа находится в режиме Только просмотр, то
пользователь может переходить по ссылкам, которые содержит документ. Переход
выполняется в новом окне браузера.



В контекстном меню доступна команда Искать
везде, при выборе которой выполняется глобальный поиск данных по выделенному
тексту. Подробнее см. раздел «Глобальный поиск»
здесь.





Совет. Чтобы скопировать
выделенный текст в поле глобального поиска можно использовать сочетание клавиш Alt + F.






С помощью команды меню Сохранить форматированный документ может быть сохранен
как текстовый файл, PDF–документ или HTML–документ.



Форматирование документа
выполняется с помощью команд командной панели форматированного документа.



Кратко перечислим предоставляемые
возможности форматирования.



Изменение
шрифта. Шрифт документа можно изменить двумя способами:



● 
выбрать команду Шрифт
и в диалоге Выбор шрифта выбрать шрифт, размер
шрифта, стиль начертания (Полужирный, Подчеркнутый, Наклонный, Зачеркнутый) и нажать кнопку OK;



● 
выделить текст, шрифт которого нужно изменить, и
с помощью соответствующих команд панели увеличить или уменьшить размер шрифта
пошагово, установить стиль шрифта (полужирный, курсив или подчеркнутый).



Выбор цвета фона
и текста. Система «1С:Предприятие» обеспечивает возможность выбрать цвет
фона и цвет текста. Для этого нужно выделить текст, на панели выбрать команду Изменение цвета фона (чтобы изменить цвет фона текста)
или Изменение цвета текста (чтобы изменить цвет
текста). В открывшемся диалоге Выбор цвета выбрать
необходимый цвет и нажать кнопку OK.



Вставка
гиперссылки. Текст форматированного документа может содержать ссылки.
Нужно выделить объект (текст или картинку), выбрать команду Вставить гиперссылку. На экран выводится диалог
формирования. В поле Ссылка указать адрес
гиперссылки и нажать кнопку ОK.



Выделенный объект форматируется в
соответствии со стилем гиперссылки. Обычно это синий цвет текста с
подчеркиванием.



Вставка
картинки. В форматированный документ можно вставлять картинки. Для
вставки картинки следует выбрать команду Вставить
картинку и указать файл в стандартном диалоге выбора файла.



Вставка
символов. Для добавления в текст символов из списка используется команда
Вставить символ.



Вставка списка. Чтобы начать
маркированный список, используется команда Маркированный
список; чтобы начать нумерованный – Нумерованный
список. Затем нужно ввести текст.



Чтобы добавить следующий элемент
списка, следует нажать клавишу Enter.



Для завершения списка нужно
нажать клавишу Enter дважды или нажать клавишу Backspace, чтобы удалить последний маркер или цифру
списка.



Настройка выравнивания. Текст
документа может быть выровнен по левому краю, по правому краю или по ширине
документа.



Нужно выделить текст, который
необходимо выровнять, и нажать кнопку Влево/Вправо/По центру/По ширине.



Текст с
отступами. Можно увеличить или уменьшить отступ от края поля для одного или
нескольких абзацев текста.



Чтобы увеличить отступ, нужно выделить текст и нажать кнопку
Увеличить отступ. Если необходимо его уменьшить, нажать
Уменьшить отступ. Также для изменения отступа
можно использовать клавиатуру:



● 
нажатие клавиши Tab –
увеличивает отступ, если курсор находится в начале абзаца, или если выделен
абзац, в ином случае добавляет символ табуляции),



● 
нажатие клавиши
Shift + Tab – уменьшает отступ.



Междустрочный
интервал. Чтобы настроить междустрочный интервал, нужно выделить текст и
выбрать команду Междустрочный интервал. В
открывшемся окне ввести размер интервала с клавиатуры или с помощью кнопок и
нажать ОК.




## 3.18. Поле планировщика



Поле планировщика предназначено для отображения и
редактирования данных планирования. В зависимости от конфигурации прикладного
решения поле может использоваться для планирования задач, расписаний, событий,
встреч и других подобных данных.



В поле
планировщика всегда присутствует одно измерение – шкала времени.






Рис. 46. Поле планировщика с периодом отображения
«День»



В зависимости от
настроек конфигурации в поле планировщика могут использоваться дополнительные измерения.
Измерения могут быть сгруппированы, а интервалы шкалы времени могут содержать
текстовые подписи и подсказки.



Набор доступных
команд определяется конфигурацией прикладного решения. Для работы с периодом отображения
поля планировщика используются команды меню формы, , расположенные в подменю Планировщик, доступном
через кнопку Меню (⋮):



● 
К началу –
переход к началу отображаемого периода.



● 
Назад –
переход к предыдущему периоду отображения. Например, при отображении периода
«День» выполняется переход к предыдущему дню.



● 
Сегодня – переход к
текущей дате.



● 
Вперед – переход к
следующему периоду отображения.



● 
К концу – переход к
концу отображаемого периода.



Элементы
управления, расположенные непосредственно на форме планировщика (кнопки,
надписи, гиперссылки), добавляются разработчиком прикладного решения и не
являются частью стандартного элемента формы Поле планировщика. Их наличие и
поведение зависят от реализации конкретной формы.







## 3.19. Поле HTML–документа



Поле HTML–документа
предназначено для просмотра HTML–документа на форме. При просмотре информации,
размещенной в поле, доступен переход по ссылкам (если такие предусмотрены),
поиск с помощью команд главного меню Правка: Найти, Найти следующий, Найти предыдущий, переход между страницами с помощью
команд контекстного меню Вперед, Назад.





Примечание. Если на поле
отображается внешний ресурс, то поиск выполняется средствами браузера.






Для данного поля доступны команды меню: Печать, Сохранить как.




## 3.20. Поле PDF–документа



Элемент формы Поле PDF–документа
предназначен для отображения содержимого PDF–документа на форме.



При загрузке документа выполняется проверка
действительности подписей (если они есть) и определение используемого стандарта
PDF.






Рис. 47.
Поле PDF–документа



Масштабирование документа. Чтобы
увеличить или уменьшить страницу, используйте команды панели инструментов поля
PDF–документа.



Для точного выбора масштаба выполните команду Меню поля (⋮) – Масштаб...

Навигация по страницам. Для перехода
по страницам используйте кнопки быстрого перехода на панели инструментов поля

Чтобы повернуть документ, используйте команды Повернуть по часовой стрелке и Повернуть

доступны в Меню формы (⋮)
–
Документ:

когда подписано).




## 3.21. Поле графической схемы



Элемент формы Поле графической схемы
предназначен для отображения и редактирования графической схемы.



Редактирование графических схем описано в приложении
«Редактор графической схемы».



Порядок обхода элементов схемы определяется при
конфигурировании.



Команда Графическая схема – Масштаб предназначена
для выбора масштаба отображения графической схемы.



Аналогичные действия по изменению масштаба можно выполнять с
помощью мыши, у которой имеется колесо прокрутки, при нажатой клавише Ctrl.



В этом поле могут отображаться общие карты маршрута бизнес–процессов
или карты маршрута конкретных экземпляров бизнес–процессов с учетом пройденных
и активных точек маршрута.



В качестве примера отображения карты маршрута в
графической схеме приведем карту маршрута бизнес–процесса продажи.






Рис. 48.
Графическая схема карты маршрута



Пройденные точки маршрута отображаются заштрихованным фоном
(на рисунке – Выписка счета). Точка маршрута
считается пройденной, если все связанные с ней задачи выполнены.



Активные точки обводятся красной пунктирной линией. Точка считается
активной, если с ней связана одна или более невыполненная задача (на рисунке – Утверждение счета).
