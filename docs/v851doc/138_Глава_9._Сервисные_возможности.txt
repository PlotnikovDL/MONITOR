# Глава 9. Сервисные возможности



Система «1С:Предприятие» предоставляет сервисные механизмы
для навигации по приложению: механизм ссылок, история работы, избранное.



Также пользователю доступна возможность использования
встроенных калькулятора, календаря, механизма для сравнения файлов и т. д.



Данная глава описывает работу с перечисленными механизмами.




## 9.1. Глобальный поиск



В «1С:Предприятие» пользователям предоставлена возможность
использовать глобальный поиск. Данный механизм
позволяет выполнять поиск по данным информационной базы, а также выполнять
другие действия, необходимые в приложении.



Чтобы начать поиск данных, на панели инструментов
нужно нажать кнопку Поиск и ввести текст в открывшемся
поле поиска главной панели приложения.






Рис. 139.
Панель поиска



Поиск начинается при нажатии клавиши Enter
или после небольшой паузы при вводе.



По умолчанию глобальный поиск осуществляет собственно поиск
по различным областям информационной базы, позволяет вычислять
арифметические выражения, а также, позволяет переходить
по навигационным ссылкам (как на объекты конфигурации, так и на
различные ресурсы сети Интернет).



Результаты поиска (не более 30 элементов) отображаются в
выпадающем списке результатов. Если в результате получено меньше 30 элементов,
выполняется поиск в данных по строке, в которой к каждому слову в веденной
строке добавляется звездочка (*) – поиск по неточному соответствию. Общее
количество результатов поиска – не более 30 элементов. Более релевантные
результаты отображаются в начале списка.



Действия системы зависят от типа введенной информации:



● 
при вводе текста будет
выполнен поиск по данным, избранному и истории работе пользователя, функциям
приложения, обсуждениям и сообщениям системы взаимодействия (если приложение
подключено к серверу системы взаимодействия), а также встроенной справочной
информации. 
Для найденного сообщения в системе взаимодействия в результате поиска
отобразится иконка группового обсуждения.

Если в прикладном решении используются хештеги (последовательности символов
после знака "#"), то можно выполнять
поиск по хештегам. При этом будут найдены все объекты с данным хештегом, а также
с хештегами, начинающимися так же, как искомый.



● 
при вставке навигационной ссылки
или внешней гиперссылки будет выполнен переход по
ссылке.



● 
при вводе выражения для вычисления (как в
формульном калькуляторе) – будет вычислено выражение. Поддерживаются все
функции работы с числами, определенные во встроенном языке. Для результата вычисления
доступны следующие действия:



● 
Скопировать –
скопировать результат в буфер обмена,



● 
Калькулятор –
открыть встроенный калькулятор, в который будет скопирован результат вычисления.



Если при выполнении поиска произошла ошибка, то в
зависимости от конфигурации, в результатах поиска отобразится информация об
ошибке либо Ошибка поиска.



В прикладном решении могут быть определены данные, при вводе
которых в поле поиска, в приложении будут выполнены действия, которые по
умолчанию выполнить нельзя. Например, в прикладном решении может быть
определена возможность создавать какие–либо объекты, если в поле поиска перед
строкой текста ввести символ +.



Поле глобального поиска можно использовать для быстрого
запуска стандартных функций приложения. Для этого нужно ввести в поле следующий
текст:



● 
избранное –
будет открыта панель избранного работы пользователя. Подробнее о работе с избранным
см. здесь.



● 
история –
будет открыта панель истории работы пользователя. Подробнее см. здесь.



● 
история сообщений –
будет открыта панель истории сообщений системы взаимодействия.



● 
календарь –
будет открыта форма календаря (Сервис и настройки – Календарь).
Подробнее см. здесь.



● 
калькулятор –
будет открыта форма калькулятора (Сервис и
настройки – Калькулятор). Подробнее см. здесь.



● 
выбор начальной страницы –
будет открыта форма выбора начальной страницы (Сервис и настройки –
Выбор начальной страницы) Подробнее см. здесь.



● 
настройки главного
меню – для открытия формы настройки главного меню (Сервис и настройки – Настройки главного меню).
Подробнее см. здесь.



● 
настройки внешнего вида
– для открытия формы настройки внешнего вида приложения (Сервис и
настройки – Настройки внешнего вида). Подробнее см. здесь.



● 
расширенные
настройки – для открытия формы расширенных настроек (Сервис и
настройки – Расширенные настройки). Подробнее см. стр. 248.



● 
о программе –
для открытия формы О программе (Сервис и настройки – О программе). Подробнее см.
здесь.



● 
обсуждения –
для перехода к панели Обсуждения (если подключена
система взаимодействия). Подробнее см. здесь.



● 
оповещения –
для открытия Центра оповещений. Подробнее см.
здесь.



● 
содержание справки –
для открытия дерева содержания справки (Сервис и настройки –
Справка – Содержание справки).



● 
справка – для
открытия справочной информации к текущей форме (Сервис и
настройки – Справка – Справка).



● 
Функции для технического
специалиста – будет открыт диалог Функции для
технического специалиста (Сервис и настройки –
Функции для технического специалиста…), если эта возможность включена
в настройках клиентского приложения. Подробнее о работе с диалогом «Функции
для технического специалиста» см. здесь.



В контекстном меню некоторых элементов формы может быть
доступна команда Искать везде <строка>. Эта
команда доступна для таблицы формы, расширения поля формы для табличного
документа, расширения поля формы для форматированного документа, поля ввода (в
том числе, поля ввода сообщения системы взаимодействия). При выборе данной
команды выделенный текст либо текст, в котором расположен курсор, будет помещен
в поле глобального поиска и будет запущен глобальный поиск. Также для начала глобального
поиска можно использовать сочетание клавиш Alt + F.




## 9.2. Работа со ссылками



В системе
предусмотрена возможность получения текстовой ссылки на любой раздел интерфейса
программы, отчет, обработку, а также на объекты информационной базы (документы,
элементы списков и т. д.).



Для разделов
интерфейса получение ссылки не поддерживается.



Полученную ссылку
можно сохранить и использовать в дальнейшем для перехода по ней. Поскольку ссылка
представляет собой текст, ее можно передать другим пользователям, например, по
электронной почте.




### 9.2.1. Получение ссылки



Для получения
ссылки в меню Меню (⋮) формы объекта следует выбрать команду Получить ссылку…
или воспользоваться командой меню Сервис и настройки – Получить ссылку.



Если фокус
установлен на строке списка, можно получить ссылку на объект текущей строки.
Для этого следует выбрать команду Получить ссылку… в контекстном меню элемента списка.



На рисунке
ниже приведен пример получения ссылки на элемент «Молоко» в списке товаров.






Рис. 140. Получение ссылки на товар «Молоко»



После
нажатия ссылки Показать ссылку открывается диалог получения ссылки.






Рис. 141. Диалог «Показать ссылку»



В диалоге в поле Ссылка на
отображается наименование объекта, для которого формируется ссылка. В диалоге
доступны следующие действия.



● 
Показать ссылку –
отображает навигационную ссылку выбранного объекта.



● 
Внешняя ссылка –
позволяет преобразовать навигационную ссылку в ссылку для внешнего использования,
например для передачи при запуске веб–клиента с открытием указанного элемента.



● 
Копировать название
– добавляет к копируемой ссылке наименование объекта данных.



● 
Копировать –
копирует текст ссылки в буфер обмена и закрывает диалог.



● 
Добавить в избранное
– помещает ссылку в Избранное (команда доступна при наличии у пользователя
права «Сохранение данных пользователя»).



Для
некоторых объектов возможно получение нескольких видов навигационных ссылок. В
этом случае в диалоге получения ссылки можно выбрать требуемый вид ссылки.






Рис. 142. Выбор вида навигационной ссылки



Если
выбран вид ссылки, для которого требуется указать представление, в диалоге
появляется поле «Введите имя ссылки», в котором можно задать или изменить
отображаемое имя ссылки.






Рис. 143. Ввод имени ссылки




### 9.2.2. Переход по ссылке



Для перехода по
ссылке можно воспользоваться глобальным поиском. Для этого в панели
инструментов следует нажать кнопку поиска, после чего откроется панель с полем
ввода. В открывшееся поле необходимо вставить текст ссылки и нажать клавишу Enter.



Также для перехода
по ссылке можно воспользоваться командой Сервис и настройки – Перейти по ссылке. В открывшемся
диалоге следует указать сохраненный ранее текст ссылки и нажать кнопку Перейти или
клавишу Enter.



Для отличия ссылок
«1С:Предприятия» от других видов ссылок (например, http или file)
используется идентификатор схемы e1c.



Если в диалог
перехода по ссылке передается внешняя ссылка «1С:Предприятия», начало которой
не совпадает с навигационной ссылкой текущей информационной базы, при переходе
выполняется попытка перехода по внутренней части ссылки, указанной после
идентификатора e1c. Если такой переход невозможен, выполняется переход по
переданной внешней ссылке.



При переходе по
ссылке пробельные символы в начале и в конце строки не учитываются.




## 9.3. Избранное



В «1С:Предприятии» избранное – это список ссылок, выбранных самим
пользователем для быстрого перехода к тем или иным разделам конфигурации,
формам объектов информационной базы, а также формам отчетов и обработок.



Для
вызова формы избранного нужно нажать кнопку Избранное панели инструментов.






Рис. 144. Кнопка «Избранное»




### 9.3.1. Добавление ссылки в Избранное



Чтобы добавить
ссылку на объект в список Избранного, воспользуйтесь одним из следующих способов:



● 
Выберите команду Добавить в избранное в меню формы
(⋮).



● 
Аналогичная команд команда доступна в меню Дополнительные
действия (⋮)
строки команды главного меню.



● 
Быстро добавить ссылку на открытую форму можно с
помощью сочетания клавиш Ctrl + D.



● 
В форме Истории нажать звездочку слева в строке
элемента.





Совет.
Можно удалить ссылку на форму из избранного, находясь в самой форме. Для этого
нажмите клавиши Ctrl + Shift + D. При этом






Если форма
объекта поддерживает несколько видов ссылок, при добавлении откроется диалог
выбора.






Рис. 145. Добавление ссылки в избранное



В нем
можно выбрать вид ссылки, при необходимости отредактировать ее имя (Имя ссылки), и
нажать Добавить.






Рис. 146. Добавление ссылки в избранное предлагает
ввести имя



После добавления
система отобразит оповещение. Если ссылка не корректна (например, относится к
внешнему приложению или к объекту, на который у пользователя нет прав), в
списке будет отображаться навигационная ссылка.




### 9.3.2. Работа с формой Избранного



Форма
избранного содержит добавленные ссылки на функции и данные, которые отображаются
в порядке важности: закрепленные элементы отображаются в начале списка.






Рис. 147. Форма избранного



Закрепление ссылки. Чтобы закрепить элемент в верхней части
списка, используйте переключатель слева от его названия. Закрепленные элементы
отображаются первыми.



В
мобильном клиенте закрепленные элементы также отображаются в главном меню
приложения.



Переименование ссылки. Чтобы изменить название, откройте меню
строки элемента (⁝) и выберите команду Переименовать,
либо нажмите клавишу F2.



Изменение порядка. Чтобы изменить положение элемента,
перетащите его выше или ниже с помощью мыши. Также можно использовать команды Переместить вверх
и Переместить
вниз в меню строки элемента (⁝) или сочетания клавиш Ctrl + Shift + Up
(перемещение вверх) и Ctrl + Shift + Dwn (перемещение вниз).



Отображение недоступных ссылок. Если ссылка относится к
объекту, недоступному пользователю или отсутствующему в текущем приложении, в
списке будет отображаться навигационная ссылка.



Удаление ссылки. Чтобы удалить элемент, откройте меню строки
(⁝)
и выберите команду Удалить.



Быстро
удалить текущую форму из Избранного можно с помощью сочетания клавиш Ctrl + Shift + D.





Примечание.
Если в Избранном есть несколько ссылок на одну и ту же форму с разными
параметрами, нажатие Ctrl + Shift + D удалит все эти
ссылки.






При
этом если в избранном были несколько ссылок на форму с разными параметрами,
будут удалены все ссылки.



Поиск в Избранном. Для поиска начните вводить название элемента
в строку поиска. В списке будут отображаться только элементы, содержащие
введенную подстроку. Совпадающие фрагменты подсвечиваются. Чтобы вернуть полный
список, очистите поле поиска.




## 9.4. История работы пользователя



«1С:Предприятие» сохраняет историю работы
пользователя, которую можно использовать для быстрого доступа к недавно
созданным или отредактированным объектам информационной базы (документам,
элементам списков и др.).



При интерактивном создании или изменении
объектов информация об этом отображается в виде оповещения и попадает в историю
(подробнее о механизме оповещений см. раздел «Оповещения» здесь).



Чтобы перейти к форме истории можно нажать
на иконку истории в главной панели. При выборе элемента списка открывается
форма объекта.






Рис. 148. Форма истории



Все элементы сгруппированы по датам. Если
одна и та же форма была открыта в разные дни, то ссылка на нее будет
присутствовать в каждом из этих дней. В пределах одного дня ссылки не
повторяются, а форма отображается с последним временем открытия.



Если элемент истории содержится в списке
избранного, рядом с ним отображается звездочка. Переключение звездочки в строке
элемента добавляет/удаляет элемент из списка избранного. Если приложение
подключено к сервису «1С:Диалог» или серверу системы взаимодействия, то
сообщения из контекстных обсуждений в истории помечаются в виде иконки диалога.



Поиск в истории. Для
поиска по истории можно использовать поле поиска, или нажать Ctrl + F, или начать ввод текста. Поиск по
истории начинается при вводе в строке поиска. При этом в списке истории
отображаются только те элементы, в представлении которых содержится введенная
строка, найденные подстроки подсвечиваются. Во время поиска элементы в списке
по дням не группируются, а рядом с элементами отображается время и дата
открытия. Для завершения поиска и возврата к полному списку истории следует
очистить строку поиска.





Примечание. Поиск не выполняется
по колонке Дата (время).







## 9.5. История данных



В «1С:Предприятии»
пользователям приложения может быть доступно хранение истории изменений объектов.
Данная возможность настраивается на этапе конфигурирования и для ограниченного
списка видов объектов. Также она может быть включена пользователем, если такая
возможность предусмотрена разработчиком программы.



Если для
объекта ведение истории используется Меню (⋮)
на форме объекта и форме списка доступна команда История изменений.






Рис. 149. Вызов истории изменений на форме объекта



При
выборе команды откроется список версий объекта. Чтобы список всегда содержал
актуальные данные, в программе должно быть настроено соответствующее обновление
данных.

Для
каждой версии в списке отображаются:



● 
№ – номер версии и
иконка вида изменения данных:



● 
Добавление (+) – объект был
создан,



● 
Изменение
("карандаш") – объект был отредактирован,



● 
Удаление ("крестик")
– объект был удален.



● 
Дата – дата
создания версии,



● 
Автор изменений –
полное имя или имя пользователя, создавшего версию,



● 
Источник изменений
– представление узла обмена данных, из которого получена версия. Колонка
отображается, если в приложении настроен обмен данными с другим приложением. Если
версия создана средствами обмена, колонка содержит строку в формате <Узел>(<План
обмена>), где <Узел> – представление узла плана обмена, <План обмена>
– представление плана обмена. Если обмен не используется, отображается текст "Это приложение".



●

комментарий к версии. Чтобы редактировать комментарий к версии, на строке

При

доступны следующие команды:



● 
Обновить – обновляет
список версий объекта данных;



●

приведет к восстановлению данных на момент создания версии. Данная команда доступна,

●

●

●

сообщение об ошибке;



●

открывает форму элемента или записи. Форма содержит текущее состояние данных в
информационной базе. В открывшейся форме данные можно записать. Запись данных
приведет к восстановлению данных на момент создания версии. Данная команда
доступна, если у пользователя установлено соответствующее право доступа;

### 9.5.2. Сравнение версий



Отчет
отображает различия двух выбранных версий данных. Отчет содержит разделы:
заголовок, поля, и табличные части.






Рис. 153. Сравнение версий объекта



Заголовок содержит
представление объекта данных на момент создания версии после изменения, для каждой
версии номер версии, дату создания, имя пользователя, который создал версию.



Далее в отчете
отображается список измененных полей. Порядок вывода полей соответствует
порядку полей в объекте. Если поля объекта не были изменены, эта часть отчета
не отображается.



Если поля нет в
версии после изменения, оно не отображается. Если поля нет в версии до
изменения и значение отличается от значения по умолчанию, то отображается
только значение версии после изменения.



Для каждой
табличной части объекта отчет отображает представление табличной части и
изменения табличной части. Если табличные части не менялись, то раздел не
отображается. Каждая строка табличной части отображается как раздел.



Поля, табличные
части, реквизиты табличной части отображаются, если у пользователя есть право
на просмотр соответствующего объекта.




#### 9.5.2.1. Используемые обозначения



Добавление
строки – Подзаголовок содержит текст Создана строка, в колонке версии
после изменения отображается номер строки в версии после изменений. Для каждого
реквизита табличной части выводится строка с указанием представления и значения
реквизита колонки данных версии после изменений.



Изменение
строки – Подзаголовок содержит текст Изменена строка, в колонке версии
после изменения отображается номер строки в версии после изменений, в колонке
версии до изменения отображается номер строки в версии до изменения. Для
каждого измененного реквизита табличной части выводится строка с указанием
представления и значениями реквизитов версий в соответствующих колонках.



Удаление
строки – Подзаголовок содержит текст Удалена строка, в колонке версии до
изменения отображается номер строки в версии до изменения. Для каждого
реквизита табличной части выводится строка с указанием представления и значения
реквизита колонке данных версии до изменения.



Перемещение
строки – Подзаголовок содержит текст Перемещена строка, в колонке версии
после изменения отображается номер строки в версии после изменения, в колонке
версии до изменения отображается номер строки в версии до изменения.




## 9.6. Оповещения



Механизм оповещений
предназначен для информирования пользователя о том, что система выполнила то
или иное действие, например, был записан или отредактирован объект. Оповещения
могут создаваться системой или разработчиком прикладного решения.

«Продажа»



●

причина оповещения.



● 
Пояснение –
уточняющая информация.



При использовании
обсуждений оповещения направляются:



● 
при добавлении сообщения в неконтекстное
обсуждение – всем участникам обсуждения, кроме автора сообщения;



● 
при добавлении сообщения в контекстное
обсуждение – всем пользователям, которых автор сообщения пожелал оповестить, а
также пользователям, включившим режим наблюдения для этого контекстного
обсуждения;



● 
при пропущенном видеозвонке – пользователю,
пропустившему видеозвонок.



Если пользователь
не ответит на видеозвонок, то ему будет направлено оповещение о пропущенном
видеозвонке:






Щелкнув мышью по
тексту оповещения, пользователь может перейти к обсуждению, из которого был
начат видеозвонок.



Если
приложение подключено к системе взаимодействия и на компьютере пользователя
установлено и запущено приложение «1С:Предприятия – оповещения и запуск», то
при появлении трех и более важных оповещений, над ними отобразится кнопка Скрыть все.
Подробнее о системе взаимодействия см. главу «Система
взаимодействия» здесь. Подробнее о приложении «1С:Предприятия – оповещения и запуск»,
см. здесь.

## 5.1. Функции для технического специалиста



Система предоставляет возможность открывать все доступные
формы и выполнять ряд сервисных действий с помощью режима «Функции для технического
специалиста».






Для перехода в этот режим следует нажать кнопку Сервис и настройки на панели инструментов и выбрать
команду Функции для технического специалиста….
Команда доступна при включенной соответствующей настройке в параметрах системы
и при наличии у пользователя права на использование данного режима (подробнее
см. раздел «Расширенные
настройки»,
здесь).






Рис. 159.
Функции для технического специалиста



В открывшемся окне отображается список объектов и функций,
доступных пользователю. В список не включаются объекты и функции, на просмотр
которых у пользователя отсутствуют права.



В группе «Стандартные» отображаются стандартные функции
системы, такие как список активных пользователей, журнал регистрации и другие.
Описание стандартных функций приводится во встроенной справке соответствующих
форм и обработок. В зависимости от реализации описание может быть доступно как
для обработки в целом, так и для отдельных форм.



Для перехода к объекту или функции следует дважды щелкнуть
по нужному элементу в дереве либо выбрать элемент в дереве и нажать кнопку Открыть форму в строке выбранного элемента.



Для быстрого поиска нужного объекта или функции в списке
можно воспользоваться строкой поиска.




## 5.2. Сообщения



При возникновении
ситуаций, на которые следует обратить внимание, в нижней части окна
отображается панель сообщений.






Работа с сообщениями аналогична
работе с сообщениями в стандартном интерфейсе. Одновременно могут отображаться
до пяти сообщений. Чтобы увидеть не поместившиеся сообщения, следует прокрутить
список с помощью клавиш Стрелка вверх/Стрелка вниз или с помощью мыши.




## 5.3. Состояние длительного процесса



Во время работы с программой могут возникать ситуации, когда
системе требуется время для выполнения некоторого процесса. Например, при
проведении большого количества документов, выполнении тяжеловесных расчетов.
При этом на экране отображается сообщение, содержащее состояние длительного
процесса.




## 5.4. Калькулятор

«1С:Предприятие» имеет встроенный калькулятор. Его можно вызвать в любом режиме
программы (кроме случаев, когда программа ждет ответа на запрос), выбрав
команду панели инструментов Сервис и настройки –
Калькулятор (или нажав сочетание клавиш Ctrl + F2).

Калькулятор включает:



●

## 5.5. Календарь



Система «1С:Предприятие» содержит встроенный
календарь. Его можно вызвать в любом режиме программы, выбрав команду главного
меню Сервис и настройки – Календарь. Календарь
позволяет вам выбирать и редактировать даты и время, а также настраивать
различные параметры для удобства работы. Вы можете использовать его для
создания событий, выбора периодов или простого просмотра дат.






Рис. 161.
Календарь в режиме выбора даты



На рисунке выше показаны основные элементы календаря:



1. Область
выбора даты – здесь можно выбрать конкретную дату.



2. 
Область ввода времени – используется для точного
указания времени.



3. 
Кнопки навигации (Вперед
и Назад) – позволяют переключаться между месяцами.



4. 
Кнопка выбора месяца – открывает выпадающий
список месяцев.



5. 
Кнопка выбора года – открывает выпадающий список
лет.



6. 
Текущее выбранное значение даты – серой рамкой
отображается выбираемая дата. Синим фоном выделяется текущая дата.



7. 
Текущее выбранное значение времени – на барабане
между синими рамками отображает выбранное время.



8. 
Ссылка Сегодня –
быстро перемещает выбор на сегодняшнюю дату.



9. 
Кнопка Подтвердить
– применяет выбранные значения.



В окне выводится календарь на
текущий месяц. Его наименование и год выводятся в верхней строке. Ниже
расположена строка дней недели.



Для смены года используется меню
выбора года, вызываемое при нажатии на текущий номер года.




### 5.5.1. Как открыть календарь



Календарь вызывается:



●

● 
Вручную – нажатием кнопки выбора даты рядом с

### 5.5.2. Как выбрать дату

## 5.7. Работа с буфером обмена



Кроме стандартных возможностей работы с буфером обмена,
предоставляемых любыми программами, работающими в среде Microsoft Windows,
«1С:Предприятие 8» предоставляет дополнительный сервис работы с числовыми
значениями в буфере обмена.



Для этого обычно используются кнопки M,
М+ и М– в заголовке
окна.



При работе во вспомогательном окне эти кнопки доступны в
заголовке формы.



Чтобы включить эту возможность, следует
активировать в меню Сервис и настройки пункт Показать М, М+, М–.






Рис. 165.
Операции с буфером обмена



Кнопка M (можно использовать
сочетание клавиш Shift + Num*) работает
аналогично стандартному действию запоминания значения в буфер обмена, но
запоминает его как число, то есть только числовые значения.



Следует иметь в виду, что для формульного калькулятора и
табло число сохраняется с региональными установками информационной базы, но без
разделителей групп и замены разделителя целой и дробной части на точку. Для
всех остальных потенциальных приемников буфера обмена число сохраняется с
региональными установками операционной системы и также без разделителей групп.
Это позволяет вставлять сохраненное в буфере обмена число как в формульный
калькулятор и табло 1С:Предприятия 8, так и в системные и офисные программы –
калькулятор Windows, Microsoft Office Excel и т. д.



Кнопка М+ (сочетание клавиш Shift + Num+) добавляет текущее значение к
значению в буфере обмена, кнопка М– (сочетание
клавиш Shift + Num–) вычитает текущее
значение из буфера обмена. Хотя собственно значение в буфере обмена хранится в
качестве строки, все указанные действия выполняются с ним как с числовым
значением.



Данные действия доступны, во–первых, во всех формах для
значений реквизитов, во–вторых, при работе с табличным документом. Кроме того,
при работе с табличным документом существует возможность выполнения
одновременных действий с несколькими ячейками: например, вы можете выделить
несколько ячеек и, используя указанные действия, поместить или добавить в буфер
обмена их сумму.



Полученные результаты в буфере обмена можно стандартным
образом использовать в любых режимах, например, для вставки значения клавишами Shift + Ins или соответствующими пунктами меню
и кнопками панелей инструментов. Разумеется, значение буфера обмена может быть
использовано и в других программах.



Текущее значение буфера обмена в числовом выражении можно
просматривать в режиме формульного калькулятора в поле М=.




## 5.8. Сравнение файлов



Режим
сравнения файлов дает возможность пользователю сравнить два любых файла. Для
сравнения файлов необходимо выбрать команду меню Сервис и настройки – Файл – Сравнить файлы. На экран будет выдан диалог для выбора
сравниваемых файлов.






Рис. 166. Диалог выбора файлов для сравнения






Рис.
167. Диалог выбора файлов для сравнения



В
полях Сравнить
и С
необходимо указать имена сравниваемых файлов. Их можно ввести вручную, выбрать
из выпадающего списка или нажатием кнопки Выбрать вызвать стандартный диалог
открытия файла и выбрать файл в этом диалоге. В выпадающем списке диалога
выбора файла можно указать любой тип исходного файла.



В
поле Как
указывается вид сравнения файлов. Выбор типа сравнения определяет, по какому
алгоритму будет производиться сравнение выбранных файлов. Допускается двоичное
сравнение. В этом случае результатом сравнения будет сообщение о совпадении или
различии файлов.



Если
в полях Сравнить
и С
указаны файлы различных типов, то выбранные файлы при сравнении будут
приводиться к виду, указанному в поле Как. Затем выполнится сравнение, и на экран будет
выведена информация о его результатах.



Если
установлен флажок Учитывать различия в разделителях строк, то при
сравнении будут учитываться символы, используемые в качестве разделителей
строк.



Для
начала сравнения нужно нажать кнопку ОK.



Если
выбран вид сравнения Текстовый документ, то выбранные файлы будут
интерпретироваться при сравнении как текстовые, несмотря на то, что в полях Сравнить и С могут быть
выбраны нетекстовые (по расширению) файлы. Если указанные файлы или один из них
не могут быть интерпретированы как текстовые, то производится двоичное
сравнение.




## 5.9. Сообщения об ошибках




## 5.10. Использование возможностей Windows 10 и Linux Unity



В операционной системе Windows 7 (и выше), а также в Linux с использованием
оболочки рабочего стола Unity,
в список переходов для «1С:Предприятие» автоматически добавляются ссылки на
запущенные ранее конфигурации. Часто используемые ссылки можно закрепить (с
помощью команды контекстного меню Закрепить в списке).



Если ярлык «1С:Предприятие» закреплен на панели
задач, то в контекстном меню также будут доступны ссылки на последние запущенные
конфигурации для быстрого запуска программы без использования специальной
программы запуска.




## 5.11. Невизуальная доступность программы



Пользователи с ограничениями по зрению могут использовать
прикладные решения на платформе «1С:Предприятие» с помощью программ экранного
доступа. С прикладным решением можно работать без использования мыши и не глядя
на экран компьютера – программа экранного доступа озвучит отображающиеся на
экране элементы интерфейса: вкладки разделов, поля форм и т.д.



Для работы рекомендуется использовать программу NVDA (распространяется
бесплатно). Данная возможность доступна для ОС Windows, начиная с версии Windows 7.
Подробнее о программе см. http://www.nvaccess.org/
и http://nvda.ru.
