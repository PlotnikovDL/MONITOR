### 1.2.2. Требования для работы Дата акселератора





Примечание. Доступно только для
лицензии КОРП. Подробнее о видах лицензий см. здесь.






Компьютер, на котором должен работать
сервис Дата акселератора, должен удовлетворять специальным требованиям:



● 
Процессор с архитектурой Intel Haswel или AMD Piledriver, а также
архитектуры процессоров Intel или AMD, разработанные после
указанных архитектур. Процессор должен иметь не менее двух ядер. Под термином
«ядро» понимается физическое ядро процессора компьютера (в том числе полученное
в результате включения технологии hyper-threading) или любое ядро
виртуальной машины, на которой работает Дата акселератор. При запуске Дата
акселератора на операционной системы, работающей под управлением
гипервизора Hyper-V, необходимо в настройках виртуальной машины выключить
флажок Выполнить перенос
на физический компьютер с другой версией процессора.



● 
64-разрядная операционная система:



● 
Семейство Microsoft Windows: 7, 8, 8.1, 10, 11
(со всеми установленными обновлениями).



● 
Семейство Microsoft Windows Server: 2012,
2012 R2, 2016, 2019, 2022.



● 
Семейство ОС Linux: совпадает с требованиями
системы «1С:Предприятие» (только версии ОС для
поддерживаемых архитектур).



● 
Оперативная память:



● 
минимальные требования ‑ 64 Гбайта.



● 
рекомендуется ‑ 512 Гбайт.



При работе под управлением ОС Linux, параметр ядра
операционной системы /sys/kernel/mm/transparent_hugepage должен быть установлен в значение
madvise
или never.
Дата акселератор при старте проверяет значение данного параметра.




## 1.3. Общие требования для клиент-серверного варианта работы




### 1.3.1. Общие требования



В
случае если кластер серверов «1С:Предприятия» и сервер баз данных
установлены на разных компьютерах, на производительность системы
может существенно влиять пропускная способность сетевого соединения между
компьютерами кластера серверов «1С:Предприятия» и компьютером сервера баз
данных. Рекомендуется использовать сетевые карты с пропускной
способностью 1 Гбит и выше для реализации следующих соединений:



● 
Компьютеры кластера серверов и компьютер,
на котором работает СУБД основной базы данных.



● 
Компьютеры кластера серверов и компьютер,
на котором работает СУБД копии базы данных ‑ если используется
механизм копии базы данных.



● 
Компьютеры кластера серверов, на котором
работает сервис Дата акселератора и остальными компьютерами, входящими
в состав кластера серверов ‑ если используется механизм Дата
акселератора.



Не
поддерживается запуск клиентского приложения на виртуальной машине, если
для сетевого адаптера виртуальной машины установлен тип подключения NAT, а кластер серверов «1С:Предприятия», с которым
должно работать клиентское приложение, расположен на компьютере, который
является хост-компьютером для виртуальной машины клиентского приложения.



При
необходимости использования некоторых механизмов системы «1С:Предприятие»,
на компьютерах с кластером серверов необходимо установить Java версии 1.8 или последующие версии. Разрядность Java должна соответствовать разрядности кластера серверов
«1С:Предприятия». Рекомендуется использовать дистрибутив Liberica. Перечень механизмов, требующих наличия Java:



● 
Утилита ring и связанные утилиты,
например, утилита лицензирования (ring license).



● 
Оптимизированный механизм реструктуризации
информационной базы (при использовании СУБД Microsoft SQL Server или PostgreSQL).



● 
Полнотекстовый поиск в данных, версия 2.



Для
корректной работы механизмов системы, Java может требовать дополнительной настройки. Описание настройки
Java см. здесь.





Важно! Для нормальной работы «1С:Предприятия»
в варианте клиент-сервер необходимо на клиентских компьютерах
отключить использование энергосберегающих режимов Sleep, Standby и Hibernate.







### 1.3.2. Права пользователя



Пользователь,
от чьего имени выполняются процессы кластера серверов (по умолчанию это
пользователь USR1CV8) должен обладать следующими правами доступа:



● 
Общие требования:



● 
Иметь полные права на каталог, указанный
в качестве каталога данных кластера. Если рабочие процессы кластера работают
от имени отдельного пользователя (с помощью файла swpuser.ini), то такому пользователю
рекомендуется запретить доступ к каталогу данных кластера.



● 
ОС Windows:



● 
Право Список содержимого папки (List folder contents)
на каталог временных файлов.



● 
Право Вход в качестве службы (Log on as a service).



● 
Право Вход в качестве пакетного задания (Log on as a batch job).



● 
Входить в группу Пользователи журналов производительности (Performance Log Users).



● 
ОС Linux:



● 
Право Чтение на каталог временных файлов.



Для
повышения безопасности кластера рекомендуется указать в файле swpuser.ini
такие настройки, чтобы рабочие процессы (rphost)
запускались от имени пользователя, который не имеет доступа к каталогу,
указанного в качестве каталога данных кластера.



Если
в системе используется файл swpuser.ini, то описание прав доступа,
которые необходимо предоставить пользователям, приведено в описании
данного конфигурационного файла (см. здесь).

Используются только библиотеки,
зарегистрированные в системном кеше динамического линковщика времени
выполнения (данную информацию можно получить, запустив команду ldconfig
-p).
Если в системе имеется несколько версий одной библиотеки, будет загружена
самая старшая версия.




## 1.4. Использование энергосберегающих режимов работы компьютера



Если на компьютере запущено
«1С:Предприятие», то использование энергосберегающих режимов будет возможно
только при соблюдении всех нижеперечисленных условий:



● 
ключ защиты установлен локально,



● 
используется файл-серверный вариант,



● 
файл базы данных расположен на локальном
диске.



В противном случае переход
в энергосберегающий режим будет запрещен независимо от вызвавшей его
причины.

### 1.5.3. Microsoft SQL Server





Внимание! Не поддерживается при
использовании кластера серверов на процессорах архитектуры ARM64 или E2K.






При установке свойства Режим совместимости (описание свойства приведено
в книге) в значение большее, чем Версия 8.3.7:



● 
Минимальной версией СУБД Microsoft SQL Server, с которой будет
работать «1С:Предприятие», становится Microsoft SQL Server 2005.



● 
Для работы системы «1С:Предприятие» требуется
установленный Microsoft OLE DB Driver for SQL Server актуальной
версии или SQL Server Native Client for SQL Server 2005 (и более старших
версий Microsoft SQL Server).



● 
Проверка наличия установленного Microsoft OLE DB Driver for SQL Server или SQL Server Native Client осуществляется при выполнении
следующих операций:



● 
создание информационной базы;



● 
загрузка информационной базы;



● 
обновление конфигурации базы данных.



Версия используемого SQL Server Native Client должна быть не младше
версии используемого Microsoft SQL Server.
