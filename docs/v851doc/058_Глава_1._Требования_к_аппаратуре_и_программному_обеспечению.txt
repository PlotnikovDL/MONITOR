# Глава 1. Требования к аппаратуре и программному обеспечению




## 1.1. Требования к аппаратуре




### 1.1.1. Общая информация



Данный раздел содержит минимальные
требования к характеристикам вычислительных систем, на которых эксплуатируется
система «1С:Предприятие».



Работа на компьютерах фирмы Apple поддерживается как
с процессорами архитектуры x86-64, так и с процессорами Apple. Работа на процессорах
Apple поддерживается
только в режиме трансляции. Режим трансляции может влиять на производительность
платформы «1С:Предприятие».



Оптимальная конфигурация оборудования
зависит не только от вида клиентского приложения платформы
«1С:Предприятие», но и от других факторов:



● 
установленного программного обеспечения,



● 
размера и числа информационных баз,



● 
числа подключенных пользователей,



● 
используемой конфигурации (прикладного решения),



● 
используемых доработок конфигурации,



● 
используемых расширений конфигураций,



● 
используемых механизмов платформы (например,
Дата акселератор) и множества других факторов.



Для получения оптимальной конфигурации
оборудования для комфортной работы не следует руководствоваться только минимальными
требованиями, а рекомендуется оценить оборудование на практическом
использовании. При выполнении внедрений в организации с большим
числом пользователей рекомендуется проводить тестирование. Полученные
результаты рекомендуется сравнивать с желаемой производительностью
и корректировать полученные требования к аппаратуре
и программному обеспечению. Для расчета параметров серверного оборудования
рекомендуется применять методику (https://its.1c.ru/db/metod8dev#content:5810).



Актуальные системные требования системы
«1С:Предприятие» с текущими уточнениями опубликованы на сайте: http://www.v8.1c.ru/requirements/.

## 1.2. Использование энергосберегающих режимов работы компьютера



Если на компьютере запущено
«1С:Предприятие», то использование энергосберегающих режимов будет запрещено,
если:



● 
используется ключ защиты программы, доступный
по сети;



● 
файл базы данных расположен на сетевом
диске (при работе в файл-серверном варианте).

## 1.5. Ограничения



Совместная работа с файловой
информационной базой возможна только клиентскими приложениями одной версии.
Архитектура (32‑ или 64-разрядное приложение) и используемая ОС (Windows или Linux) клиентского
приложения, выполняющего доступ к файловой информационной базе, не
ограничивается.



Максимальное количество одновременных
соединений с одной файловой информационной базы равно 1024.



Совместная работа с файловой
информационной базой по сети поддерживается только для сетевых ресурсов,
доступ к которым осуществляется по протоколу SMB (CIFS). Такие ресурсы могут
располагаться как на компьютерах с ОС Windows, так и на
компьютерах с ОС Linux.



Клиентское приложение, работающее под
управлением ОС macOS, не поддерживает работу
с информационной базой, расположенной на сетевом ресурсе.



При работе системы следует помнить, что
разрядность внешних компонент и COM-объектов (при работе под
управлением ОС Windows) должны совпадать
с разрядностью и типом используемого приложения системы
«1С:Предприятие» (тонкое или толстое клиентское приложение, сервер или
веб-браузер). Другими словами, если какой-то COM-объект существует только
в 32-разрядном варианте, то его будет нельзя использовать
в 64-разрядной версии системы «1С:Предприятие».



В том случае,
когда предполагается использование системы «1С:Предприятие»
на вычислительных комплексах, основанных на процессорах архитектуры ARM64, следует помнить о следующих ограничениях:



● 
Не поддерживается работа под управлением ОС Windows.



● 
Работа на процессорах Apple поддерживается только
в режиме трансляции.



● 
Не поддерживается использование аппаратных
ключей защиты.



● 
Не поддерживается автоматическое обновление
клиентского приложения.



В
том случае, когда предполагается использование системы «1С:Предприятие»
на вычислительных комплексах, основанных на процессорах архитектуры E2K, следует помнить о следующих ограничениях:



● 
Не поддерживается работа под управлением ОС Windows.



● 
Не поддерживается использование аппаратных
ключей защиты.



● 
Не поддерживается автоматическое обновление
клиентского приложения.

### 1.6.3. Виды лицензий




#### 1.6.3.1. Общая информация



При
использовании «1С:Предприятия» (кроме учебной версии) требуются различные
лицензии. Лицензии требуются для работы различных элементов системы. В это
разделе описываются возможные варианты лицензий системы «1С:Предприятие».




#### 1.6.3.2. Клиентская лицензия



Назначение:
требуется для функционирования клиентского приложения в любом
из вариантов использования: файловый или клиент-серверный варианты.



Виды:



● 
Клиентская лицензия КОРП.
Позволяет использовать все возможности системы.



● 
Клиентская лицензия ПРОФ.
Позволяет использовать возможности системы, в соответствии
с ограничениями серверной лицензии ПРОФ.



● 
Клиентская лицензия
«1С:Аналитика». Клиентская лицензия «1С:Аналитика» (подробнее см. здесь) позволяет
использовать систему «1С:Аналитика» одновременно нескольким пользователям (один
сеанс ‑ один пользователь). Количество одновременных сеансов не
может превышать номинал лицензии. Клиентская лицензия системы «1С:Аналитика»
может использоваться с любой серверной лицензией системы «1С:Предприятие».



Если в информационной базе одновременно
обслуживаются не более 5 сеансов клиентских приложений, то технически
и лицензионно допускается запуск 1 сеанса системы «1С:Аналитика»
без наличия клиентской лицензии «1С:Аналитики».



При использовании серверной лицензии
«1С:Предприятия» уровня ПРОФ, клиентская лицензия «1С:Аналитики» позволяет
использовать механизм копий базы данных (подробнее см. здесь) совместно

Накладывает на используемую систему следующие ограничения:

пользователь (подробнее см. здесь).



● 
Не поддерживается редактирование конфигурации
с помощью конфигуратора.



● 
Не поддерживается работа в клиент-серверном
варианте (подробнее см. здесь).



● 
Не поддерживается работа в распределенной
информационной базе (подробнее см. здесь).



● 
Не поддерживается предоставление
интернет-сервисов (подробнее см. здесь).



● 
Не поддерживается работа в режиме
веб-клиента.



● 
Не поддерживается работа в режиме внешнего
соединения.



● 
Не поддерживается работа с расширениями

● 
Использование системы «1С:Предприятие»
на компьютерах с процессорами архитектуры E2K.

«1С:Предприятие» (клиентскому или серверному). Лицензия расширения может

#### 1.6.3.6. Лицензия для разработчиков (community версия)



Назначение:
позволяет осуществлять полноценную разработку на платформе
«1С:Предприятие» без необходимости покупки лицензий. Платформа, которая
использует активированную лицензию для разработчиков, не может быть использована
в коммерческих системах.



Информация:
https://1c.ru/news/info.jsp?id=30506.



Особенности
лицензии для разработчиков:



● 
Может быть использована только с системой
«1С:Предприятие» версии 8.3.23 и последующих версий.



● 
Существует только в виде программной
лицензии.



● 
Может быть одновременно активирована на 1
клиентский и 1 серверный компьютер (даже если это разные компьютеры).



● 
Ограничена по сроку действия: 1 неделя.
Обновление лицензии выполняется автоматически, при наличии подключения
к сети Интернет.



● 
Лицензии для разработчиков не суммируются.



● 
Если к информационной базе подключены
сеансы, которые используют лицензии ПРОФ или КОРП, то к одной информационной
базе может быть одновременно подключено не более 3 (включительно) клиентских
сеансов, получивших одну лицензию для разработчиков.



● 
Используется в следующих случаях:



● 
Для файловой информационной базы: в том
случае, если не обнаружена какая-либо другая лицензия (базовая, ПРОФ или КОРП).



● 
Для клиент-серверной информационной базы:
в том случае, если не обнаружена какая-либо другая лицензия (ПРОФ или
КОРП). Лицензия для разработчиков может быть использована одновременно
и как клиентская и как серверная.



● 
Собирается статистика использования
и отправляется в Центр Контроля Обновлений. Передается статистика
использования механизмов платформы, пользовательские данные информационной базы
не передаются. Статистика собирается с компьютера разработчика
из приложения, которое использует активированную лицензию для разработчиков.



Лицензия
для разработчиков разрешает:



● 
Обеспечивать совместную работу в одной
информационной базе для следующих сеансов (лицензии раздаются сервером
«1С:Предприятия»):



● 
Не более 3 сеансов с прямым подключением.



● 
Не более 1 сеанса при подключении через веб-сервер.



● 
Не более 1 сеанса клиента системы
«1С:Аналитики».



● 
Не более 1 сеанса мобильного клиента.



● 
Использовать функциональность лицензии типа
КОРП, если с информационной базой работает не более 10 лицензируемых
сеансов.



Лицензия
для разработчиков не разрешает:



● 
Коммерческое использование лицензии.



● 
Доступ к механизмам, которые требуют
лицензию для тестирования выделенной функциональности.



● 
Привязку лицензии к ключу аппаратной
защиты.



● 
Использование более 2 разных лицензий для
разработчиков в одной информационной базе.



● 
Использование более 1 рабочего сервера
в кластере:



● 
При добавлении рабочего сервера формируется
ошибка.



● 
Если в кластере уже создано более одного
рабочего сервера, то будет использоваться только один рабочий сервер (произвольный).



● 
Использование клиентских лицензии ПРОФ или КОРП
для доступа к кластеру серверов, который использует в качестве серверной
лицензии лицензию для разработчиков.




#### 1.6.3.7. Лицензия для тестирования выделенной функциональности



Назначение:
требуется для проведения тестирования механизмов, для которых заявлен статус
бета-тестирования (выделенной функциональности). Лицензия для тестирования
выделенной функциональности является дополнением к клиентской или серверной
лицензии любого вида.



Информация:
http://1c.ru/news/info.jsp?id=25182.



Особенности
лицензии для тестирования выделенной функциональности:



● 
Лицензия активируется на компьютер:



● 
Кластера серверов «1С:Предприятия».



● 
Расширения веб-сервера, которое работает
с файловым вариантом информационной базы, в которой используется выделенная
функциональность.



● 
Клиентского приложения, которое работает
с файловым вариантом информационной базы, в которой используется
выделенная функциональность.



● 
Лицензия может раздаваться сервисом
лицензирования кластера серверов.



● 
Лицензии для тестирования выделенной
функциональности должны быть:



● 
В клиент-серверном варианте: в количестве,
не меньшем, чем количество серверных лицензий кластера серверов,
на котором выполняется тестирование.



● 
В файловом варианте: хотя-бы одна лицензия.

● 
Возможности, использование которых требуют
лицензии для тестирования выделенной функциональности, определяется при выпуске

● 
Окончательные лицензионные требования
тестируемых механизмов определяются в момент вывода этих механизмов

направить запрос по электронной почте betaplatform@1c.ru.




### 1.6.4. Применимость лицензии КОРП



Все
возможности системы, включая доступные под лицензией КОРП, кроме Дата
акселератора, доступны в том случае, если к каждой информационной
базе кластера серверов «1С:Предприятие» подключено одновременно не более 10
клиентских сеансов. И серверная и все клиентские лицензии в этом
случае могут быть уровня ПРОФ. При подключении хотя бы к одной из баз
кластера серверов 11-го сеанса, этот сеанс получит ошибку, если в кластере
серверов используются возможности, доступные под лицензией КОРП,
а лицензий КОРП недостаточно.



Если
в системе используются возможности, предоставляемые лицензией КОРП,
и количество сеансов пользователей одной информационной базы превышает 10,
то начало нового сеанса пользователя этой информационной базы возможно только
при наличии достаточного количества клиентских и серверных лицензий КОРП.



Количество
лицензий КОРП считается достаточным, если все используемые кластером серверные
лицензии имеют уровень КОРП и дополнительно выполняется одно
из следующих требований:



1. Клиентское
приложение получило клиентскую лицензии КОРП.



2. Клиентское
приложение получило лицензию ПРОФ и в момент подключения нового
клиентского соединения выполняется следующее соотношение используемых лицензий:
УП <= 1,1 * ЛК. В данном
выражении:



● 
УП ‑ количество
уникальных пользователей (уникальность отслеживается по именам
пользователей) клиентских сеансов кластера с любыми видами лицензий,
включая начинающийся сеанс;



● 
ЛК ‑ количество
клиентских лицензий КОРП, доступных для выдачи сервером «1С:Предприятия» (вне
зависимости от фактического состояния флага выдачи лицензий сервером
«1С:Предприятия»).



Рассмотрим
пример. На компьютер с сервером «1С:Предприятия» установлена серверная
лицензия КОРП и клиентская лицензия КОРП на 50 пользователей. Данная
конфигурация обеспечивает:



● 
Произвольное количество клиентских подключений,
если каждое клиентское подключение обладает лицензией уровня КОРП.



● 
Подключение к серверу клиентского
приложения с лицензией ПРОФ будет возможно в том случае, если после
подключения сервер станет обслуживать сеансы не более 1,1*50 = 55 уникальных
пользователей информационной базы. Таким образом, пользователь с лицензией
ПРОФ сможет получить доступ к информационной базе только в том
случае, если, в момент подключения, количество уникальных пользователей
кластера серверов (считая стартующий сеанс) не превышает 55.



Техническая возможность входа в систему, использующую
возможности КОРП, незначительной части пользователей с лицензиями ПРОФ (из
общего количества пользователей с лицензиями КОРП), сделана для снижения
опасности отказа в обслуживании значительного количества пользователей
из-за случайного недосмотра или ошибки. Работа пользователей с лицензиями
уровня ПРОФ в системе уровня КОРП не является правомерной.
